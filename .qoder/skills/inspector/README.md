# 🎯 巡检宝项目 - 督察Inspector Skill

> **项目规范守护者** - 确保所有Agent遵守规则

---

## 📋 概述

Inspector是巡检宝项目的"纪律委员"，负责：
- 🚨 实时监控代码规范违规
- 🔒 检测安全漏洞
- 🎯 防止AI幻觉行为
- 📊 管理上下文长度
- 📝 提供修正建议

### 工作模式

**混合模式**（主动 + 被动）：
- **主动模式**: 定期巡查（任务开始、关键节点、上下文过长）
- **被动模式**: 按需调用（Agent自查、完成后检查、决策前检查）

## 🎯 核心功能

### 1. 违规检测规则库

**总计18条规则**，分为4大类：

#### 🚨 P0永久禁止（5条）
| 规则ID | 违规类型 | 处理 |
|--------|---------|------|
| P0-001 | 硬编码敏感信息 | 立即中断 |
| P0-002 | 未验证就断言 | 立即中断 |
| P0-003 | 循环内查数据库 | 立即中断 |
| P0-004 | 未备份改数据 | 立即中断 |
| P0-005 | 凭空捏造路径 | 立即中断 |

#### ⚠️ P1代码质量（5条）
| 规则ID | 违规类型 | 处理 |
|--------|---------|------|
| P1-001 | any类型 | 要求修正 |
| P1-002 | SELECT * | 要求修正 |
| P1-003 | 未处理error | 要求修正 |
| P1-004 | 未释放资源 | 要求修正 |
| P1-005 | 隐式返回类型 | 要求修正 |

#### 📝 P2流程规范（4条）
| 规则ID | 违规类型 | 处理 |
|--------|---------|------|
| P2-001 | 跳过代码审查 | 建议修正 |
| P2-002 | 缺少日志 | 建议修正 |
| P2-003 | 任务状态未更新 | 建议修正 |
| P2-004 | 上下文过长 | 必须修正 |

#### 🔒 安全漏洞（4条）
| 规则ID | 违规类型 | 处理 |
|--------|---------|------|
| SEC-001 | SQL注入 | 立即中断 |
| SEC-002 | XSS风险 | 要求修正 |
| SEC-003 | 敏感信息泄露 | 要求修正 |
| SEC-004 | 不安全依赖 | 要求修正 |

---

### 2. 检测执行流程

#### 快速检查（<5秒）
适用：日常编码即时反馈
范围：最近1-2个代码块 + P0违规

#### 标准检查（15-30秒）
适用：文件完成后的全面检查
范围：整个文件 + P0,P1,P2规则

#### 深度检查（1-5分钟）
适用：关键任务最终审查
范围：整个模块 + 全量规则 + 安全扫描

---

### 3. 警告分级机制

#### 🚨 CRITICAL（P0）
- 图标：🚨
- 模板：红色背景
- 要求确认：是
- 阻塞继续：是

#### ⚠️ WARNING（P1）
- 图标：⚠️
- 模板：黄色背景
- 要求确认：否
- 阻塞继续：否

#### 📝 NOTICE（P2）
- 图标：📝
- 模板：蓝色背景
- 要求确认：否
- 阻塞继续：否

## 🚀 快速开始

### 调用时机

**推荐调用点**：
1. 🎯 开始新任务前
2. 📝 完成代码编写后


**代码检查**：
```

```

**上下文管理**：
```
inspector context-status
```

## 🎓 使用指南

### 场景1: 日常编码

**流程**：
1. 开始编码
2. 完成一个函数
3. 调用 `inspector quick` 快速检查
4. 修正P0/P1违规
5. 继续下一个函数

### 场景2: 文件完成

**流程**：
1. 完成整个文件
2. 调用 `inspector check <file>` 标准检查
3. 修正所有P0违规
4. 修正所有P1违规
5. 考虑采纳P2建议
6. 提交代码审查

### 场景3: 复杂任务

**流程**：
1. 开始任务前调用 `inspector context` 检查上下文
2. 如果L3+，先生成摘要
3. 关键节点调用 `inspector check` 检查
4. 任务完成后调用 `inspector deep` 深度检查
5. 确认所有违规已修正
6. 发起PR审查

## ⚙️ 自定义配置

### 启用/禁用规则
在 `rules_library.json` 中：
```json
{
  "rules": [
    {
      "id": "P1-001",
      "enabled": true,
      "severity_override": "NOTICE"
    }
  ]
}
```

### 添加自定义规则
```json
{
  "id": "CUSTOM-001",
  "name": "自定义规则",
  "detection": {
    "patterns": ["your_pattern"]
  },
  "severity": "WARNING",
  "action": "require_fix"
}
```

## 🚨 常见问题

### Q1: 规则太严格了怎么办？
A1: 可以调整规则库配置，降低某些规则的严重级别

### Q2: 遇到规则冲突怎么办？
A2: 使用豁免注释（见templates.md）

### Q3: 误报了怎么办？
A3: 使用误报申诉模板（见templates.md）

### Q4: 需要临时违反规则怎么办？
A4: 申请豁免（见templates.md）
