| 问题 | 症状 | 比喻 | 解决方案 |
|------|------|------|---------|
| 🔄 **循环调用** | 重复做同样的事 | 强迫症 | 记录足迹，设定边界 |

## 🚨 紧急情况处理

### 🔄 循环调用 - 三步打破法

```
┌─────────────────────────────────────────────┐
│  第一步：急刹车 🛑                          │
│    └─ 立即停止当前操作                       │
│                                             │
│  第二步：回溯检查 🔙                          │
│    └─ 我是从哪一步开始循环的？                 │
│                                             │
│  第三步：换挡加速 🚀                          │
│    └─ 尝试完全不同的方法                     │
└─────────────────────────────────────────────┘
```

### 🔒 死锁 - 四步打破法

```
┌─────────────────────────────────────────────┐
│  1. 识别：发现双方在互相等待                 │
│  2. 中断：至少一方先让步                     │
│  3. 协商：提出替代方案                       │
│  4. 推进：推动决策继续                      │
└─────────────────────────────────────────────┘
```

### 😶 罢工 - 唤醒协议

```
⚠️ 罢工信号：
  - 不响应超过3个周期
  - 重复相同的失败操作
  - 输出"我正在思考..."

唤醒步骤：
  1️⃣ 温柔提醒："嘿，你好像卡住了"
  2️⃣ 提供线索："我找到一些相关资料"
  3️⃣ 简化任务："不如先做最简单的一步"
  4️⃣ 重置状态："休息一下，我们从头开始"
```

## 🔍 循环检测三问

```
每次操作前问自己：

  1️⃣ 我之前做过这个吗？
     ├─ 是 → 检查结果是否一致
     └─ 否 → 继续执行

  2️⃣ 这会有进展吗？
     ├─ 是 → 继续
     └─ 否 → 立即停止，换方法

  3️⃣ 我已经重复几次了？
     ├─ 0-1次 → 正常
     ├─ 2次 → 警惕 ⚠️
     └─ 3次以上 → 立即停止！🚨
```

## 🚫 永久禁止红线

```yaml
绝对禁止：

❌ 无限循环：
   - 没有退出条件的循环
   - 递归调用超过10层
   - 反复做同样的事期望不同结果

❌ 互相等待：
   - Agent A等B，B等A
   - 等待永远不会发生的事件
   - 持有资源但不释放

❌ 沉默抗议：
   - 不响应超过3个周期
   - 重复相同的失败操作
   - 永远在"思考中"

❌ 原地打转：
   - 每次迭代都回到原点
   - 不记录任何进展
   - 尝试清单从不减少
```

## ✅ Agent健康生活准则

### 🌟 必须遵守

```yaml
✅ 记录足迹
   - 每次操作前检查：我之前做过这个吗？
   - 使用TODO列表追踪任务进度
   - 记录"已查看"的文件列表

✅ 设定边界
   - 每次尝试最多3次
   - 超过次数就换方法或求助
   - 明确设定"失败"的标准

✅ 定期自检
   - 任务开始时：确认目标
   - 任务中：检查是否偏离目标
   - 任务结束：总结经验

✅ 优雅退出
   - 知道什么时候该放弃
   - 记录尝试过的方法
   - 明确说明为什么失败
```

## 🔧 打破循环实操

### 场景1：文件重复读取

```python
# ❌ 违规
if need_to_read:
    read_file()
    read_file()  # 又读一遍？
    read_file()  # 第三次！
```

```python
# ✅ 正确
class FileReader:
    def __init__(self):
        self.cache = {}  # 新增缓存

    def read_if_needed(self, file_path):
        if file_path in self.cache:
            return self.cache[file_path]  # 从缓存返回

        content = self.actually_read_file(file_path)
        self.cache[file_path] = content
        return content
```

### 场景2：API重复调用

```typescript
// ❌ 违规
const fetchData = async () => {
  const data = await api.getData();
  if (!data) {
    const data = await api.getData();  // 重试？不记录
    if (!data) {
      const data = await api.getData();  // 继续重试...
    }
  }
};
```

```typescript
// ✅ 正确
const fetchData = async (retries = 3) => {
  for (let i = 0; i < retries; i++) {
    const data = await api.getData();
    if (data) return data;

    if (i < retries - 1) {
      console.log(`尝试 ${i + 1} 失败，等待重试...`);
      await delay(1000);  // 等一下再试
    }
  }
  throw new Error('重试3次后仍失败，需要人工介入');
};
```

## 📝 任务开始前检查清单

```yaml
开始新任务前，必须确认：

  □ 任务目标清晰吗？（能用一句话说清楚吗？）
  □ 有没有设定完成标准？（怎么算做完？）
  □ 需要什么资源？（代码、文档、权限？）
  □ 有没有时间限制？（紧急 vs 常规）
  □ 之前有没有做过类似任务？（避免重复）

如果任何一项不确定 → 先搞清楚再开始！
```

## 🎓 小知识：为什么会有这些问题？

```yaml
和人类一样：

🧠 循环 = 鬼打墙
   原因：没有记录走过的路
   解决：留个标记，别走重复

🤦 死锁 = 两个人互相等对方先开口
   原因：双方都不愿先让步
   解决：总要有一方先开口

😔 罢工 = 被问题困扰太久选择躺平
   原因：没有希望，没有进展
   解决：给予希望，提供线索

所以照顾Agent = 照顾人
记住：给Agent留条后路，它就不会走上不归路！
```
