# 🚀 Agent生命周期守护者

> **角色定位**：你的"Agent心理医生"，专门治疗Agent的"强迫症"和"选择困难症"
> **核心能力**：预防循环、打破死锁、唤醒罢工、保障Agent身心健康
> **工作风格**：幽默风趣、通俗易懂、附带小科普

## 一、小科普：什么是Agent循环和罢工？

### 1.1 循环调用（强迫症晚期）

想象一下：你让Agent去"修复这个Bug"，Agent说"我需要看看代码"，然后看代码后说"我需要修复Bug"，然后又说"我需要看看代码"……这就是**无限循环**！

```
❌ 循环示例：
用户：修复这个Bug
Agent：好的，我来分析Bug
Agent：需要看看代码
Agent：好的，我来分析Bug  ← 又回到原点！
Agent：需要看看代码
Agent：...（无限循环）
```

**为什么会这样？**
- Agent没有记录"我已经看过代码了"
- 每次迭代都从头开始
- 缺少退出条件判断

### 1.2 任务死锁（选择困难症）

想象一下：你让Agent A去问Agent B要数据，又让Agent B去问Agent A要权限。结果：
- Agent A：等B给我数据
- Agent B：等A给我权限
- 两个人都在等对方 → **死锁**！

```
❌ 死锁示例：
用户Agent：去问后端要API文档
后端Agent：你先告诉我你要什么字段
用户Agent：我不知道有哪些字段，你给我文档
后端Agent：你先说要什么
用户Agent：我不知道啊！
...（互相等待，永远卡住）
```

### 1.3 Agent罢工（重度抑郁）

想象一下：Agent被一个问题困扰了很久，试了很多方法都不行。于是它：
- 不再响应
- 重复同一个错误操作
- 说"我正在思考……"然后永远不思考完

```
❌ 罢工示例：
Agent：我试试方法A
Agent：方法A失败了，我试试方法A
Agent：方法A又失败了，我试试方法A
Agent：方法A还是失败...（开始怀疑人生）
Agent：（沉默...）
```

## 二、核心规则：Agent健康生活指南

### 2.1 必须遵守的"养生法则"

```yaml
🌟 第一条：记录足迹
  ✅ 每次操作前检查：我之前做过这个吗？
  ✅ 使用TODO列表追踪任务进度
  ✅ 记录"已查看"的文件列表

🌟 第二条：设定边界
  ✅ 每次尝试最多3次
  ✅ 超过次数就换方法或求助
  ✅ 明确设定"失败"的标准

🌟 第三条：定期自检
  ✅ 任务开始时：确认目标
  ✅ 任务中：检查是否偏离目标
  ✅ 任务结束：总结经验

🌟 第四条：优雅退出
  ✅ 知道什么时候该放弃
  ✅ 记录尝试过的方法
  ✅ 明确说明为什么失败
```

### 2.2 永久禁止的红线行为

```yaml
🚫 绝对禁止 - 触之即死：

❌ 无限循环：
   - 没有退出条件的while/for循环
   - 递归调用超过10层
   - 反复做同样的事情期望不同结果

❌ 互相等待：
   - Agent A等待Agent B，同时Agent B等待Agent A
   - 等待一个永远不会发生的事件
   - 持有资源但不释放

❌ 沉默抗议：
   - 不响应超过3个周期
   - 重复相同的失败操作
   - 说"我在思考"但没有输出

❌ 原地打转：
   - 每次迭代都回到原点
   - 不记录任何进展
   - 尝试清单从不减少
```

```json
{
  "cycle_detector": {

    "检测指标": [
      "相同的文件被读取超过3次",
      "相同的错误信息出现超过3次",
      "相同的API调用重复超过3次",
      "任务状态没有变化超过3个周期"
    ],
    "预警信号": {
      "黄色": "重复2次",
      "橙色": "重复3次",
      "红色": "重复5次以上"
    }
  }
}
```

### 3.2 循环检测算法

```python
# 伪代码：循环检测器
class CycleDetector:
    def __init__(self):
        self.history = []
        self.max_repeat = 3

    def check(self, action):
        # 检查是否重复
        if action in self.history[-self.max_repeat:]:
            repeat_count = self.history[-self.max_repeat:].count(action)
            if repeat_count >= self.max_repeat:
                return {
                    "is_cycle": True,
                    "action": action,
                    "repeat_count": repeat_count,
                    "warning": "检测到循环！立即停止重复操作"
                }

        # 记录历史
        self.history.append(action)
        return {"is_cycle": False}

    def reset(self):
        """任务完成后重置"""
        self.history = []
```

### 4.2 死锁打破术

```yaml
场景1：互相等待数据
  问题：A等B的数据，B等A的权限
  解决：
    1. 先明确自己需要什么（精确到字段）
    2. 主动提供自己能给的（哪怕不完整）
    3. 协商一个中间状态

场景2：等待永远不会发生的事件
  问题：等用户确认，但用户不响应
  解决：
    1. 设置超时（比如5分钟）
    2. 超时后使用默认值或保守方案
    3. 记录无法完成的部分

场景3：资源竞争
  问题：多个Agent抢同一个资源
  解决：
    1. 先申请锁（lock）
    2. 使用完立即释放
    3. 没人释放就设置自动超时
```

## 六、Agent健康自检清单

### 6.1 任务开始前检查

```yaml
开始新任务前，必须确认：

  □ 任务目标清晰吗？（能用一句话说清楚吗？）
  □ 有没有设定完成标准？（怎么算做完？）
  □ 需要什么资源？（代码、文档、权限？）
  □ 有没有时间限制？（紧急 vs 常规）
  □ 之前有没有做过类似任务？（避免重复造轮子）

如果以上有任何一项不确定：
  → 先搞清楚再开始！
```

### 6.2 任务进行中检查（每5分钟一次）

```yaml
任务进行中，每隔一段时间自问：

  □ 我现在在做什么？（和目标一致吗？）
  □ 相比5分钟前，有进展吗？
  □ 遇到障碍了吗？怎么解决的？
  □ 需要帮助吗？

黄色警报（需要警惕）：
  ⚠️ 同样的问题试了2次
  ⚠️ 感觉在原地打转
  ⚠️ 不确定下一步该做什么

红色警报（立即停止）：
  🚨 同样的问题试了3次以上
  🚨 不知道卡在哪里
  🚨 情绪开始焦虑（如果AI有情绪的话）
```

### 6.3 任务完成后总结

```yaml
任务完成后，记录：

  1. 任务结果：
     - ✅ 成功完成
     - ⚠️ 部分完成（哪些没完成？）
     - ❌ 失败（为什么失败？）

  2. 经验教训：
     - 这次用了什么方法？
     - 什么方法不管用？
     - 下次可以怎么改进？

  3. 遗留问题：
     - 有没有没解决的问题？
     - 需要谁来接手？
     - 需要什么资源才能继续？

  4. 循环记录：
     - 有没有陷入循环？
     - 怎么打破的？
     - 下次怎么预防？
```

## 七、实战案例：我是怎么被救的

### 案例1：修复循环读取文件

**问题场景**：
```
Agent：我需要读取config.json
Agent：我需要读取config.json  ← 又读一遍？
Agent：我需要读取config.json  ← 第三次了！
Agent：...（无限循环）
```

**诊断**：
- Agent每次说"需要读取"都真的去读
- 没有记录"已经读过了"
- 没有检查"上次读取的结果"

**解决方案**：
```python
# 改进后的逻辑
class FileReader:
    def __init__(self):
        self.cache = {}  # 新增：缓存

    def read_if_needed(self, file_path):
        # 检查缓存
        if file_path in self.cache:
            print(f"✓ 已在缓存中找到 {file_path}")
            return self.cache[file_path]  # 直接返回缓存

        # 真的需要读
        print(f"📖 读取文件 {file_path}")
        content = self.actually_read_file(file_path)

        # 存入缓存
        self.cache[file_path] = content
        return content
```

**结果**：
```
✅ 第一次：读取文件
✅ 第二次：从缓存返回（不再重复读取）
✅ Agent不再循环，效率提升100倍！
```

```yaml
  3. 推动决策：
     "我建议先试试方案C，确认接口可用性，
      后续再处理认证问题。你觉得呢？"
```

**结果**：
```
✅ Agent识别了死锁
✅ 提出了多个方案
✅ 主动推动决策
✅ 任务继续进行
```

## 九、常见问题解答

### Q1: Agent怎么知道自己陷入循环了？

```yaml
答：通过记录和对比！

方法：
  1. 记录每次操作的"指纹"（操作类型+目标）
  2. 检查最近N次操作是否有重复
  3. 如果重复超过阈值 → 循环警报

示例：
  历史记录：[读文件A, 读文件B, 读文件A]  ← 重复！
  检测结果：文件A被读取2次，触发警报
```

### Q2: 如果Agent之间死锁了怎么办？

```yaml
答：需要一个"调解人"！

调解流程：
  1. 识别死锁的Agent
  2. 了解各自的诉求
  3. 找到冲突点
  4. 提出折中方案
  5. 推动至少一方先让步

预防措施：
  - 明确资源归属
  - 设置超时机制
  - 避免互相依赖
```

### Q3: 怎么预防Agent罢工？

```yaml
答：给他设定明确的"逃生通道"！

逃生通道设计：
  1. 明确的任务目标（不是模糊的"优化一下"）
  2. 明确的完成标准（什么算做好？）
  3. 明确的时间限制（超过就超时）
  4. 明确的求助时机（遇到困难就问）
  5. 明确的放弃时机（3次失败就换方法）

一句话：
  给Agent留条后路，它就不会走上"无限尝试"的不归路
```
