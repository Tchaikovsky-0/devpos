# code-review-expert

**Description:** 代码审查专家 - 自动代码质量检查、安全漏洞扫描、性能优化建议
**Trigger:** 当用户需要代码审查、质量检查、安全审计或性能优化时

## 概述

代码审查专家是一个智能代码质量守护神，能够自动检测代码问题、提供优化建议、生成质量报告。

## 核心能力

### 1. 代码规范检查
- ESLint 规则检查
- Prettier 格式检查
- 命名规范验证
- 代码复杂度分析

### 2. 安全漏洞扫描
- 常见安全漏洞检测
- 依赖包安全审计
- 敏感信息泄露检查
- 输入验证漏洞识别

### 3. 性能优化建议
- 算法复杂度分析
- 内存使用优化
- 网络请求优化
- 渲染性能建议

### 4. 架构设计审查
- 模块耦合度分析
- 设计模式应用检查
- 接口设计合理性
- 可扩展性评估

## 使用示例

### 示例1：快速代码审查
```
用户：帮我审查这个文件
智能体：自动运行代码审查，生成详细报告
```

### 示例2：安全审计
```
用户：检查项目安全漏洞
智能体：扫描依赖包、检查代码安全、生成安全报告
```

### 示例3：性能优化
```
用户：优化这个组件的性能
智能体：分析性能瓶颈、提供优化方案、验证优化效果
```

## 工具集成

- **ESLint**: 代码规范检查
- **Prettier**: 代码格式化
- **Security Scanner**: 安全漏洞扫描
- **Performance Analyzer**: 性能分析
- **Test Coverage**: 测试覆盖率检查

## 输出格式

### 审查报告格式
```markdown
# 代码审查报告

## 质量评分: 85/100

### ✅ 通过检查
- 代码规范符合要求
- 无安全漏洞

### ⚠️ 需要改进
- 性能优化建议
- 测试覆盖率不足

### 🔧 具体建议
1. 优化算法复杂度
2. 增加单元测试
3. 改进错误处理
```

## 配置选项

### 审查级别
- **基础**: 仅检查语法和规范
- **标准**: 包含安全和性能检查
- **全面**: 包含架构和最佳实践

### 输出格式
- **简洁**: 仅显示关键问题
- **详细**: 包含具体建议和示例
- **报告**: 生成完整审查报告

## 最佳实践

1. **定期审查**: 建议每次提交前运行代码审查
2. **渐进改进**: 优先修复高风险问题
3. **团队协作**: 将审查结果分享给团队成员
4. **持续学习**: 根据审查结果改进编码习惯

## 故障排除

### 常见问题
- **依赖缺失**: 确保安装了必要的审查工具
- **配置错误**: 检查审查配置文件
- **权限问题**: 确保有足够的文件访问权限

### 性能优化
- 使用缓存机制减少重复分析
- 并行处理多个文件
- 增量审查只检查变更部分
