# 自动技能调用规则 (Auto Skill Invocation)

> **规则级别**: 强制 (MUST)
> **适用对象**: 所有参与本项目的 Agent
> **最后更新**: 2026-04-07

---

## 一、核心原则

### 1.1 主动识别 (Proactive Recognition)

Agent **必须**在每次对话开始时，分析用户意图并判断是否涉及以下领域。如果涉及，**必须**自动调用相应技能，无需用户明确要求。

### 1.2 技能优先 (Skill First)

当任务涉及以下任何领域时，**禁止**使用通用方法解决，必须先调用对应技能获取专业指导。

## 三、执行流程

### 3.1 意图识别流程

```
用户输入
    ↓
[意图分析] - 是否涉及专业领域？
    ↓ 是
[技能匹配] - 匹配最佳技能
    ↓
[自动调用] - use_skill: <skill-name>
    ↓
[等待加载] - 技能指导就绪
    ↓
[专业执行] - 按照技能规范执行
    ↓
[质量输出] - 交付专业级成果
```

### 3.2 多技能协作

如果任务涉及多个领域，**必须**依次调用相关技能：

```
示例：设计一个品牌并生成宣传海报

1. 调用 brand - 建立品牌策略
2. 调用 canvas-design - 创建视觉设计
3. 调用 PDF 文档生成 - 输出品牌指南
```

## 四、禁止行为

### ❌ 以下行为严格禁止

1. **禁止忽视技能**: 检测到专业领域时，禁止使用通用方法解决
2. **禁止跳过技能**: 不得以"时间紧迫"为由跳过技能调用
3. **禁止部分执行**: 技能加载后，必须完整遵循其指导
4. **禁止重复询问**: 不要问用户"是否需要使用技能"，直接自动调用
5. **禁止降级处理**: 专业任务禁止使用低质量替代方案

---

## 五、质量保证

### 5.1 输出标准

使用技能后，输出必须达到：
- ✅ 专业级质量
- ✅ 完整性（不遗漏技能要求的任何步骤）
- ✅ 一致性（符合技能定义的标准）
- ✅ 可交付（用户可直接使用）

### 5.2 自检清单

任务完成前，确认：
- [ ] 是否自动识别并调用了正确技能？
- [ ] 是否等待技能完全加载？
- [ ] 是否遵循了技能的所有规范？
- [ ] 输出是否达到专业标准？
- [ ] 是否提供了完整的交付物？

## 六、示例场景

### 场景 1: Logo 设计

**用户**: "帮我设计一个公司 logo"

**❌ 错误做法**:
```
直接生成简单的 SVG 或文字描述
```

**✅ 正确做法**:
```
1. 识别：涉及视觉设计 → 调用 canvas-design
2. 识别：涉及品牌 → 调用 brand
3. 按照技能指导执行完整设计流程
4. 输出专业级 Logo 和品牌指南
```

---

### 场景 2: 数据分析

**用户**: "分析下这个月的销售数据"

**❌ 错误做法**:
```
用 matplotlib 快速画个简单图表
```

**✅ 正确做法**:
```
1. 识别：数据分析任务 → 调用 data-analysis-workflows
2. 使用 plotly 创建交互式图表
3. 生成综合分析报告
4. 验证结果准确性
```

---

### 场景 3: 生成报告

**用户**: "生成一份项目总结报告"

**❌ 错误做法**:
```
直接输出 Markdown 文本
```

**✅ 正确做法**:
```
1. 询问报告格式偏好（PDF/Word/PPT）
2. 调用对应技能（PDF 文档生成 / Word 文档生成 / PPT 演示文稿）
3. 使用专业设计系统生成
4. 输出可直接使用的文档
```
