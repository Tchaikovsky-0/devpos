## 一、产品概述

### 1.1 产品定位

**巡检宝（XunjianBao）** — 面向重工业企业的智能监控平台

```
一句话定位：
"面向重工业企业的智能监控平台，通过 OpenClaw AI Agent 和 YOLO 检测，
让监控从'被动观看'升级为'主动思考'。"
```

### 1.2 目标用户

| 用户群体   | 特征             | 设计关注点     |
| ------ | -------------- | --------- |
| 重工业企业  | 国企、政府背景，讲究专业稳重 | 信息密度、效率优先 |
| 高校/研究所 | 学术严谨，注重数据可追溯   | 清晰层级、证据完整 |
| 无人机飞手  | 技术导向，追求高效操作    | 快捷键、批量操作  |

### 1.3 核心场景

```
用户每日核心工作流：

  打开系统 ──► 查看监控 ──► 发现异常 ──► 处理告警 ──► 生成报告
     │           │            │            │            │
     ▼           ▼            ▼            ▼            ▼
  监控中心    数据大屏      AI检测       告警中心      AI助手
```

## 二、设计理念

### 2.1 核心关键词

> **精密 · 秩序 · 力量 · 克制**

### 2.2 设计哲学

巡检宝代表着工业监控领域的最高标准——不仅是技术的精确，更是视觉的克制与力量的平衡。

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   "政府、国企、高校以及无人机飞手这些用户群体，                    │
│    他们需要的是一种不喧哗、不媚俗、沉稳有力的视觉体验。"          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 设计原则

| 原则       | 说明           | 设计表现                 |
| -------- | ------------ | -------------------- |
| **场景闭环** | 一个模块完成一个完整场景 | 监控中心 = 查看 + 检测 + 回放  |
| **功能内聚** | 相关功能不要分散     | AI对话/分析/报告 → 统一为智能协同 |
| **深度优先** | 核心功能做到行业最佳   | 告警处理要有工作流引擎          |
| **减少选择** | 不要让用户思考用哪个功能 | 去掉重复入口               |

### 2.4 功能收敛

```
从 30+ 个分散的浅层功能
        ↓
收敛为 4 个深度打磨的核心模块
```

| 原功能（已下线）                              | 整合到  |
| ------------------------------------- | ---- |
| Dashboard, Gallery, Sensors, Device   | 监控中枢 |
| AI Chat, Analysis, Reports, Knowledge | 智能协同 |
| AlertCenter, Workflow, Tasks          | 告警处置 |
| Users, Permissions, Tenants, Settings | 资产设备 |

## 三、设计系统

### 3.1 三档明暗灰度主题

#### 深境模式 (Deep Mode) ⭐推荐

适合 24/7 监控室环境，减少眼部疲劳，让视频画面成为绝对主角。

```
背景色系：
┌──────────────────────────────────────┐
│ ██ #0D1117  bg-primary (深海背景)    │
│ ██ #161B22  bg-secondary (卡片)      │
│ ██ #21262D  bg-tertiary (悬停)      │
│ ██ #30363D  border (边框分隔线)      │
└──────────────────────────────────────┘

文字色系：
┌──────────────────────────────────────┐
│ ██ #E6EDF3  text-primary (主文字)    │
│ ██ #7D8590  text-secondary (次要)    │
│ ██ #484F58  text-tertiary (禁用)     │
└──────────────────────────────────────┘
```

#### 均衡模式 (Balanced Mode)

日常工作的理想选择，明暗平衡，适合办公室和分析场景。

```
背景：#1E2228 / #2A3038 / #3A424A / #3D444D
文字：#E8EDF2 / #9CA3AF / #6B7280
```

#### 清境模式 (Clear Mode)

明亮通透，适合演示和汇报场景，展现专业与开放。

```
背景：#F5F7FA / #FFFFFF / #EDF0F4 / #DDE2E8
文字：#1F2328 / #656D76 / #8C959F
```

## 六、交互规范

### 6.1 导航交互

#### 侧边栏导航

```
默认状态: 图标 + 文字 (桌面端) / 仅图标 (移动端)
Hover: 背景变亮，显示 tooltip
Active: 左侧 2px 强调色指示条 + 背景高亮
点击: 路由跳转，保持 layout 不刷新
```

#### 模块切换

```
点击导航项 → 平滑过渡到对应页面
当前模块在 Header 显示模块标题和状态
```

### 6.2 列表交互

#### 列表项选择

```
默认: 透明背景
Hover: bg-tertiary
选中: bg-accent/10 + 边框 accent/30
点击: 高亮选中项，右侧详情面板更新
```

#### 列表滚动

```
左侧列表: 独立滚动区域，不影响其他区域
虚拟滚动: 列表项超过 100 条时启用
```

### 6.3 表单交互

#### 输入框

```
Focus: 边框变蓝 (accent)
Blur: 触发验证
Error: 显示错误提示
Success: 显示成功提示
```

#### 过滤器

```
单选: 胶囊形按钮组，选中项高亮
多选: Checkbox 组
搜索: 带搜索图标的输入框
```

### 6.4 快捷操作

```
Cmd/Ctrl + K: 唤起 OpenClaw 面板
点击 AI 按钮: 唤起 OpenClaw 面板
Escape: 关闭弹窗/面板
```

## 七、动效规范

### 7.1 过渡时间

```
--duration-fast: 150ms      /* 按钮、链接 */
--duration-normal: 200ms    /* 卡片、面板 */
--duration-slow: 300ms      /* 模态框、抽屉 */
```

### 7.2 缓动函数

```
--ease-default: cubic-bezier(0.4, 0, 0.2, 1);
--ease-in: cubic-bezier(0.4, 0, 1, 1);
--ease-out: cubic-bezier(0, 0, 0.2, 1);
--ease-spring: cubic-bezier(0.34, 1.56, 0.64, 1);
```

### 7.3 常用动画

#### 淡入

```css
@keyframes fadeIn {
  from { opacity: 0; }
  to { opacity: 1; }
}
```

#### 滑入

```css
@keyframes slideIn {
  from {
    opacity: 0;
    transform: translateX(20px);
  }
  to {
    opacity: 1;
    transform: translateX(0);
  }
}
```

#### 缩放弹出

```css
@keyframes scaleIn {
  from {
    opacity: 0;
    transform: scale(0.95);
  }
  to {
    opacity: 1;
    transform: scale(1);
  }
}
```

### 7.4 特殊动效

#### 告警闪烁

```css
@keyframes alertPulse {
  0%, 100% { opacity: 1; }
  50% { opacity: 0.5; }
}
```

#### 加载旋转

```css
@keyframes spin {
  from { transform: rotate(0deg); }
  to { transform: rotate(360deg); }
}
```

## 八、响应式策略

### 8.1 断点

```
--breakpoint-sm: 640px   /* 手机横屏 */
--breakpoint-md: 768px   /* 平板 */
--breakpoint-lg: 1024px  /* 小桌面 */
--breakpoint-xl: 1280px  /* 桌面 */
--breakpoint-2xl: 1536px /* 大桌面 */
```

### 8.2 布局适配

| 断点      | 布局变化             |
| ------- | ---------------- |
| < md    | 侧边栏 → 底部 TabBar  |
| md - lg | 侧边栏压缩为 64px 图标模式 |
| >= lg   | 侧边栏完整显示          |

### 8.3 网格适配

```
桌面端 (>1024px): 使用完整三栏布局
        2xl:grid-cols-[280px_minmax(0,1fr)_340px]

平板端 (768-1024px): 两栏布局
        md:grid-cols-3 (指标卡片)
        2xl:grid-cols-[minmax(0,1fr)_360px]

手机端 (<768px): 单栏堆叠
        网格变单列
        侧边栏变为底部导航
```

### 8.4 视频网格适配

```
桌面端: 4x4 (16画面)
笔记本: 3x3 (9画面)
平板: 2x2 (4画面)
手机: 1x1 (1画面)
```
