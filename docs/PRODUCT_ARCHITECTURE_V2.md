# 巡检宝产品架构 2.0

> **设计原则**: 聚焦核心、深度打磨、场景闭环
> 
> **目标**: 从"功能堆砌"转向"价值驱动"

---

## 一、核心洞察

### 1.1 用户主路径分析

```
用户每天的核心工作流:

    打开系统 ──► 查看监控 ──► 发现异常 ──► 处理告警 ──► 生成报告
       │           │            │            │            │
       ▼           ▼            ▼            ▼            ▼
    监控中心    数据大屏      AI检测/      告警中心      AI助手
                              人工发现

关键洞察:
- 80% 时间花在"查看监控"和"处理告警"
- AI 应该在"发现异常"和"生成报告"环节发力
- 其他功能都是支撑这三个核心场景的
```

### 1.2 功能收敛原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **场景闭环** | 一个模块完成一个完整场景 | 监控中心 = 查看 + 检测 + 回放 |
| **功能内聚** | 相关功能不要分散 | AI对话/分析/报告 → 统一为 AI助手 |
| **深度优先** | 核心功能做到行业最佳 | 告警处理要有工作流引擎 |
| **减少选择** | 不要让用户思考用哪个功能 | 去掉重复入口 |

**定位**: 用户每天打开的第一个页面，所有监控相关功能的统一入口

**核心功能**:

```
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                  视频墙 (Video Wall)                 │   │
│  │  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐                   │   │
│  │  │ 1x1 │ │ 2x2 │ │ 3x3 │ │ 4x4 │  布局切换          │   │
│  │  └─────┘ └─────┘ └─────┘ └─────┘                   │   │
│  │                                                     │   │
│  │  [视频1] [视频2] [视频3] [视频4] ...               │   │
│  │   🟢AI    🟡     🔴告警   🟢AI                     │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐     │
│  │   设备列表    │  │   时间轴     │  │   AI检测面板  │     │
│  │  (左侧面板)   │  │  (底部面板)  │  │  (右侧面板)   │     │
│  └──────────────┘  └──────────────┘  └──────────────┘     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**深度特性**:
- **智能布局**: 根据设备重要性自动推荐布局
- **AI叠加**: 视频上直接显示检测框和标签
- **时间轴**: 支持拖拽回放，标记事件点
- **设备健康度**: 每个设备显示在线率、故障率
- **一键轮巡**: 设置轮巡规则，自动切换画面

**整合的原功能**:
- ✅ 数据大屏 (Dashboard)
- ✅ 视频设备 (Sensors)
- ✅ 设备状态 (Device)
- ✅ 媒体库回放 (Gallery回放部分)
- ✅ 地图视图 (LiveMap - 作为子视图)

## 六、关键设计决策

### 6.1 为什么从 19 个入口收敛到 4 个？

**用户视角**:
- 打开系统 → 看监控 (监控中心)
- 发现问题 → 问 AI (AI助手)
- 确认问题 → 处理告警 (告警中心)
- 需要配置 → 资产管理

**产品视角**:
- 每个模块都是一个完整的场景闭环
- 减少用户的选择成本
- 集中资源做深核心功能

### 6.2 如何处理功能之间的跳转？

**场景化跳转**:
```
监控中心发现异常 ──► 一键唤起 AI 助手分析
                        │
                        ▼
AI 助手确认告警 ──► 一键创建告警工单
                        │
                        ▼
告警中心处理完成 ──► 一键生成报告
```

**设计原则**:
- 不打破用户当前场景
- 支持快速返回
- 保持上下文连贯

### 6.3 如何保证深度？

**每个核心模块都要有**:
1. **行业最佳实践**: 参考一线监控产品 (如海康、大华)
2. **自动化能力**: 减少人工操作
3. **智能化特性**: AI 赋能，提升效率
4. **数据分析**: 不只是展示，要有洞察

## 七、成功指标

|------|------|------|------|
| 平均找到功能时间 | 30s | 5s | 导航简化 |

**总结**: 从"大而全"转向"小而美"，用 4 个深度打磨的核心模块，替代 30+ 个分散的浅层功能。
