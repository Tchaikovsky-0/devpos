# 巡检宝 UI 界面布局优化方案

> 版本: 1.0 | 日期: 2026-04-07 | 待决策

---

## 一、现状诊断

### 1.1 全局布局 (Layout.tsx)

**当前结构：**

```
┌────┬───────────────────────────────────────┐
│侧栏 │ 顶栏（标题 + 状态标签 + 操作按钮）      │
│76px ├───────────────────────────────────────┤
│图标 │                                       │
│导航 │        内容区 <Outlet />               │
│    │        padding: 24px                    │
│    │                                       │
│    │                                       │
└────┴───────────────────────────────────────┘
     + OpenClaw 浮动面板
```

**诊断出的问题：**

| 编号 | 问题 | 影响 | 严重度 |
|------|------|------|--------|
| L1 | 顶栏占用固定垂直空间（约80px），在监控中枢这种需要最大化视频画面的场景中浪费了宝贵的纵向像素 | 视频网格有效面积减少约12% | 高 |
| L2 | 侧边栏宽度固定76px，仅图标无文字标签，依赖 hover 弹出卡片显示模块名，认知负荷高 | 用户需要逐个 hover 才能确认模块位置，学习成本高 | 中 |
| L3 | 8个导航模块全部平铺，无分组逻辑，"航拍图库"和"媒体库"的关系不明确 | 模块数量多时找目标慢 | 中 |
| L4 | 没有面包屑导航，也没有页面级别的二级导航 | 在深层内容中无法快速定位 | 低 |
| L5 | 顶栏中"待处置告警"和"智能协同贯穿工作流"是静态信息，在所有页面都展示，与页面无关 | 信息噪声 | 中 |
| L6 | OpenClaw面板和ContextActionStrip两个入口重叠，操作逻辑重复 | 用户困惑该点哪里 | 中 |

### 1.2 监控中枢 (Center.tsx)

**当前结构：**

```
┌─────────────────────────────────────────────┐
│ ContextActionStrip（3个AI操作按钮）             │
├─────────────────────────────────────────────┤
│ WorkspacePanel                                │
│ ┌──────────────────────────────────────────┐ │
│ │ [全部] [无人机] [固定监控]    6在线·2需关注 │ │
│ │ ┌────────┬─────────────────────────────┐ │ │
│ │ │焦点信息  │ 布局: [1][4][9][16][自] [沉浸]│ │ │
│ │ ├────────┼─────────────────────────────┤ │ │
│ │ │画面列表  │      视频网格                │ │ │
│ │ │220px   │      StreamGrid              │ │ │
│ │ │        │                              │ │ │
│ │ └────────┴─────────────────────────────┘ │ │
│ └──────────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
```

**问题：**
- ContextActionStrip 占用约48px垂直空间，但实际使用频率低
- 画面列表 220px 固定宽度，在 9/16 画面模式下过于拥挤
- "焦点信息栏"和"布局控制栏"合并为一行，信息混杂
- 沉浸模式仍显示在顶层 overlay，而非全屏接管

### 1.3 告警处置 / 任务协同 / 资产设备 / 智能协同

这四个页面共享同一个布局模式：

```
┌─────────────────────────────────────────────┐
│ ContextActionStrip（标题+摘要+3个AI按钮）      │
├─────────────────────────────────────────────┤
│ [MetricTile] [MetricTile] [MetricTile]       │
├────────────────────────────┬────────────────┤
│ WorkspacePanel             │ aside 360px   │
│ SectionHeader              │ WorkspacePanel│
│ 事件流/执行链/设备台账/...   │ 详情面板       │
│ (ListItem列表)             │ 关联上下文      │
│                           │ 协同说明       │
└────────────────────────────┴────────────────┘
```

**问题：**
- ContextActionStrip 在每个页面重复，约48px高度
- MetricTile 行在四个页面都存在，但数值不同，属于页面级摘要
- 右侧 aside 360px 在非宽屏下挤压左侧列表空间
- 四个页面的详情面板结构相似但代码不共享

### 1.4 媒体库 (Media.tsx)

**问题：**
- 1428行，三种模式（缺陷案例/图片库/资料目录）代码交织
- 每种模式各自有搜索、筛选、批量操作逻辑
- 图片库的网格视图和列表视图切换体验不一致

## 二、参考标杆分析

### 2.1 大疆司空 2 (DJI FlightHub 2)

| 特性 | 描述 | 可借鉴点 |
|------|------|---------|
| 虚拟座舱 | 全屏接管视频画面，所有控件浮动叠加 | 沉浸模式应全屏接管，而非 overlay |
| 一站式面板 | 左侧设备列表 + 中间视频墙 + 右侧任务信息 | 三栏式监控布局 |
| 告警气泡 | 告警以浮动气泡形式出现在视频画面上 | 告警不应只在独立页面，应在视频上叠加 |
| 任务航线 | 可视化航线编辑和任务下发 | 任务直接绑定到视频画面 |

### 2.2 SCADA/HMI 设计标准 (ISA-101)

| 原则 | 描述 | 可借鉴点 |
|------|------|---------|
| 四级显示层次 | 总体概览 → 区域概览 → 设备详情 → 诊断分析 | 页面之间的层级关系应有清晰的信息密度递进 |
| 40-60%留白 | 避免信息过载 | 当前页面信息密度偏高 |
| 渐进披露 | 概览屏只显示关键指标，详情屏才展示全部 | 告警列表/任务列表的摘要 vs 详情 |
| 色彩策略 | 正常用灰度，异常用彩色 | 减少不必要的状态颜色 |
| 3次点击原则 | 从概览到任何操作不超过3次点击 | 评估当前导航路径深度 |

### 2.3 Grafana 仪表盘

| 特性 | 描述 | 可借鉴点 |
|------|------|---------|
| 可变面板网格 | 面板大小自由调整，自动排列 | 视频网格布局的灵活性 |
| 顶部统计行 | 关键指标一行展示 | MetricTile 的排列方式 |
| Row 分组 | 逻辑相关面板分组，可折叠 | 页面内区域的分组 |

### 2.4 通用监控软件共性

| 共性 | 说明 |
|------|------|
| 视频占最大化面积 | 监控场景下视频画面应尽可能大 |
| 告警与视频融合 | 告警直接出现在关联的视频画面上，而非独立页面 |
| 左侧设备树 | 设备列表通常在左侧，支持搜索和过滤 |
| 底部时间轴 | 回放场景下底部有统一的时间轴 |
| 右键菜单操作 | 视频画面右键触发操作菜单 |

## 三、优化方案（3个方案供选择）

---

### 方案 A：「纯净监控台」— 专注视频最大化

**核心理念**：以监控中枢为核心，所有其他功能围绕视频画面展开，不离开监控上下文。

#### 全局布局变更

```
┌────┬───────────────────────────────────────────┐
│侧栏 │ 顶栏简化为单行：模块名 + 面包屑 + 用户操作    │
│76px ├───────────────────────────────────────────┤
│    │                                           │
│图标 │         内容区 <Outlet />                 │
│导航 │         无额外 padding                     │
│    │         （各页面自行管理间距）                │
│    │                                           │
│    │                                           │
│底部 ├───────────────────────────────────────────┤
│状态栏│ 告警计数 | 设备在线 | WS连接 | AI状态 | 时间  │
└────┴───────────────────────────────────────────┘
```

**变更项：**

| 编号 | 变更 | 理由 |
|------|------|------|
| A-L1 | 顶栏从约80px缩减为48px单行 | 释放32px给内容区 |
| A-L2 | 移除顶栏中的"待处置告警"和"智能协同"静态标签 | 移入底部状态栏 |
| A-L3 | 新增底部状态栏（24px），集中展示全局运行状态 | 参考 IDE 底部状态栏模式 |
| A-L4 | 侧边栏增加分组分隔线：监控类/管理类/AI类 | 降低认知负荷 |
| A-L5 | 删除所有页面的 ContextActionStrip 组件 | 统一移入全局 OpenClaw 面板 |

#### 监控中枢优化

```
┌─────────────────────────────────────────────────┐
│ [全部][无人机][固定监控]   布局:[1][4][9][16] [沉浸] │
├────────┬────────────────────────────────────────┤
│设备列表  │                                        │
│可折叠   │           视频网格（最大化面积）           │
│搜索+过滤 │                                        │
│        │         告警气泡叠加在视频上               │
│        │                                        │
│        │                                        │
│        │                                        │
├────────┴────────────────────────────────────────┤
│ 选中画面信息栏（可收起）                            │
└─────────────────────────────────────────────────┘
```

**变更项：**
- 删除 ContextActionStrip，AI操作移入视频画面的右键菜单
- 设备列表改为可折叠侧栏（200-280px），默认展开
- 视频网格占满剩余空间，无内边距
- 选中画面信息栏移到视频下方，可收起
- 沉浸模式全屏接管（隐藏顶栏、侧栏、底栏）

#### 告警/任务/资产/协同页面优化

```
┌─────────────────────────────────────────────────┐
│ 页面标题 + 面包屑 + 搜索框 + 筛选器                │
├─────────────────────────────────────────────────┤
│ [MetricTile] [MetricTile] [MetricTile] [Metric] │
├──────────────────────────────┬──────────────────┤
│ 列表区域（flex-1）            │ 详情面板          │
│                              │ 可折叠            │
│                              │ 280-400px         │
│                              │                  │
└──────────────────────────────┴──────────────────┘
```

**变更项：**
- ContextActionStrip 替换为"智能操作"浮动按钮（FAB），点击弹出操作菜单
- MetricTile 紧跟标题行，不额外占行
- 详情面板改为可折叠抽屉，默认展开，可收起

#### 优劣势评估

| 维度 | 评估 |
|------|------|
| 视频面积 | 提升 15-20% |
| 导航效率 | 3次点击内可达任何页面 |
| 一致性 | 高（所有页面共享统一框架） |
| 改动量 | 中等（Layout + Center 大改，其他页面小改） |
| 风险 | 底部状态栏是新增概念，需要用户适应 |

### 方案 B：「指挥中心」— 信息密度优先

**核心理念**：保留当前整体框架，通过微调间距、折叠次要信息来优化，不改变大的布局结构。

#### 全局布局变更

```
┌────┬───────────────────────────────────────────┐
│侧栏 │ 顶栏：模块名 + 状态标签 + 全局操作            │
│76px ├───────────────────────────────────────────┤
│    │                                           │
│    │         内容区 <Outlet />                 │
│    │         padding: 16px（从24px缩减）          │
│    │                                           │
│    │                                           │
└────┴───────────────────────────────────────────┘
```

**变更项：**

| 编号 | 变更 | 理由 |
|------|------|------|
| B-L1 | 内容区 padding 从 24px 缩减为 16px | 增加8%有效面积 |
| B-L2 | 顶栏中"智能协同贯穿工作流"标签删除 | 减少信息噪声 |
| B-L3 | 侧边栏 hover 弹出卡片增加模块描述 | 提升可发现性 |
| B-L4 | 告警计数从顶栏移到侧边栏告警图标 badge | 精简顶栏 |

#### 监控中枢优化

```
┌─────────────────────────────────────────────────┐
│ [全部][无人机][固定监控]   布局:[1][4][9][16][自] │
├─────────────────────────────────────────────────┤
│ ┌──────────┬──────────────────────────────────┐ │
│ │ 画面列表   │          视频网格                 │ │
│ │ 可拖拽宽度 │     StreamGrid                   │ │
│ │ min:160px │                                  │ │
│ │ max:320px │                                  │ │
│ │          │                                  │ │
│ └──────────┴──────────────────────────────────┘ │
└─────────────────────────────────────────────────┘
```

**变更项：**
- ContextActionStrip 折叠为单行紧凑模式（24px），只显示标题和展开按钮
- "焦点信息栏"删除，改为选中画面时在列表项高亮显示详情
- 画面列表改为可拖拽宽度（160-320px）
- 沉浸模式保持当前 overlay 方式，但扩大 inset

#### 其他页面

- ContextActionStrip 保留但折叠为紧凑模式
- MetricTile 行保持不变
- 右侧详情面板宽度从 360px 缩小为 320px

#### 优劣势评估

| 维度 | 评估 |
|------|------|
| 视频面积 | 提升 5-8% |
| 导航效率 | 与当前一致 |
| 一致性 | 高（改动小） |
| 改动量 | 小（微调为主） |
| 风险 | 低 |

### 方案 C：「融合工作台」— 打破页面边界

**核心理念**：借鉴大疆司空2和 VS Code，所有功能在同一个工作台内通过面板组合展示，不做页面切换。

#### 全局布局

```
┌────┬───────────────────────────────────────────────────┐
│侧栏 │ 活动栏 + 标签栏                                     │
│76px │ ┌─────┐┌─────┐┌─────┐                             │
│    │ │监控中枢││告警  ││任务  │   （可同时打开多个标签）      │
│    │ └─────┘└─────┘└─────┘                             │
│    │ ┌──────────────────┬───────────────┬──────────────┐ │
│    │ │                  │               │              │ │
│    │ │    主面板         │   侧面板       │  AI面板      │ │
│    │ │  （视频/列表/...） │  （详情/上下文） │（OpenClaw）   │ │
│    │ │                  │  可拖拽宽度     │ 可折叠       │ │
│    │ │                  │               │              │ │
│    │ └──────────────────┴───────────────┴──────────────┘ │
│    │ ┌──────────────────────────────────────────────────┐ │
│    │ │ 底栏：告警 | 设备 | WS | 时间                      │ │
│    │ └──────────────────────────────────────────────────┘ │
└────┴───────────────────────────────────────────────────┘
```

**变更项：**

| 编号 | 变更 | 理由 |
|------|------|------|
| C-L1 | 引入标签栏（Tab Bar），支持多页面同时打开 | 无需切换页面即可查看告警+视频 |
| C-L2 | 侧面板（右1）可拖拽，展示详情/上下文 | 替代各页面的 aside 区域 |
| C-L3 | AI面板（右2）常驻可折叠，替代 OpenClaw overlay | 智能协同成为一等公民 |
| C-L4 | 底栏展示全局状态 | 释放顶栏空间 |
| C-L5 | 删除 ContextActionStrip | 统一入口到 AI 面板 |

#### 优劣势评估

| 维度 | 评估 |
|------|------|
| 视频面积 | 与方案A相当 |
| 导航效率 | 最高（多标签并行） |
| 一致性 | 最高（统一面板框架） |
| 改动量 | 大（Layout 需重写，所有页面需适配） |
| 风险 | 高（架构变动大，回归测试多） |

## 四、跨方案建议（不论选哪个方案都建议执行）

以下改进与方案选择无关，属于独立优化项：

| 编号 | 建议 | 理由 |
|------|------|------|
| G1 | 侧边栏分组：监控类（中枢/媒体/图库）、管理类（告警/任务/资产/系统）、AI类（智能协同） | 降低认知负荷 |
| G2 | 告警信息融合到视频画面：在 StreamGrid 每个视频格子上叠加告警角标 | 无需切换页面即可感知告警 |
| G3 | 全局快捷键系统：Esc退出沉浸、Cmd+K打开AI、数字键切换布局 | 提升操作效率 |
| G4 | 详情面板组件化：提取通用 DetailDrawer 组件，告警/任务/资产/协同复用 | 减少代码重复 |
| G5 | 媒体库拆分：将 Media.tsx 拆为3个独立子页面（缺陷案例/图片库/资料目录），通过Tab切换 | 降低单文件复杂度 |
| G6 | 统一圆角：消除所有 `rounded-[18px]`/`rounded-[22px]`/`rounded-[24px]` 等硬编码值 | 与之前的设计系统统一工作对齐 |

## 五、推荐与决策点

### 我的推荐

**方案 A（纯净监控台）** 是推荐方案。

理由：
1. 以视频最大化为核心，符合巡检宝"监控平台"的产品定位
2. 改动量适中（约3-5天开发），风险可控
3. 底部状态栏是成熟模式（VS Code/Figma/Slack 均采用）
4. 为后续方案C（融合工作台）打下基础

### 需要你拍板的决策点

| 决策点 | 选项 | 我的建议 |
|--------|------|---------|
| **主方案选择** | A / B / C | A |
| **底部状态栏** | 要 / 不要 | 要（方案A的核心特征） |
| **ContextActionStrip** | 保留并优化 / 删除并整合到AI面板 | 删除并整合 |
| **设备列表宽度** | 固定 / 可拖拽 | 可拖拽（160-320px） |
| **详情面板** | 固定右侧 / 可折叠抽屉 | 可折叠抽屉 |
| **侧边栏分组** | 保持平铺 / 增加分组 | 增加分组 |
| **媒体库拆分** | 保持单文件 / 拆为3个子页面 | 拆为3个子页面 |
| **跨方案通用项** | 全部执行 / 选择执行 | 全部执行（G1-G6） |
