# 监控大屏模块 Spec

## Why
巡检宝需要一个专业的多画面监控大屏，支持同时显示多个视频流（最多20+路），让监控人员能够实时掌握全局态势。这是产品的核心功能，也是展示OpenClaw AI能力的重要窗口。

## What Changes
- 新增监控大屏页面 `/monitor`
- 实现多画面布局切换（1x1、2x2、3x3、4x4、5x5、自定义布局）
- 支持无人机视频流专区（固定6块）
- 支持其他视频流动态展示
- 集成OpenClaw AI助手，实现智能监控
- 实现视频流状态监控和告警提示
- 支持视频流拖拽排序和布局保存

## Impact
- Affected specs: 视频流接入规范、前端性能规范、OpenClaw集成规范
- Affected code: 
  - 新增 `/app/src/routes/Monitor.tsx` (监控大屏主页面)
  - 新增 `/app/src/components/monitor/` (监控组件目录)
  - 新增 `/app/src/api/stream.ts` (视频流API)
  - 修改 `/app/src/router.tsx` (添加路由)

## ADDED Requirements

### Requirement: 多画面布局系统
系统应提供灵活的多画面布局功能，支持用户在不同布局间快速切换。

#### Scenario: 切换布局模式
- **WHEN** 用户点击布局切换按钮
- **THEN** 系统应立即切换到对应的布局模式（1x1、2x2、3x3、4x4、5x5）
- **AND** 所有视频流应平滑过渡到新布局
- **AND** 当前播放的视频流应保持播放状态

#### Scenario: 自定义布局
- **WHEN** 用户选择自定义布局
- **THEN** 系统应允许用户手动调整每个视频窗口的大小和位置
- **AND** 用户可以保存自定义布局配置
- **AND** 下次打开时自动应用保存的布局

### Requirement: 无人机视频流专区
系统应为无人机视频流提供独立的展示区域，固定显示6路无人机画面。

#### Scenario: 无人机专区展示
- **WHEN** 用户打开监控大屏
- **THEN** 系统应在左侧或顶部显示无人机专区（固定6块）
- **AND** 无人机视频流应始终可见，不随布局切换而隐藏
- **AND** 每个无人机窗口应显示设备名称、电量、信号强度等状态信息

#### Scenario: 无人机视频流异常
- **WHEN** 某路无人机视频流断开连接
- **THEN** 系统应在对应窗口显示"信号丢失"提示
- **AND** 自动尝试重连（最多3次）
- **AND** 在状态栏显示告警信息

### Requirement: 普通视频流管理
系统应支持动态管理其他视频流（摄像头、录像回放等）。

#### Scenario: 视频流列表展示
- **WHEN** 用户打开监控大屏
- **THEN** 系统应根据当前布局自动填充视频流
- **AND** 视频流应按照优先级排序（在线 > 离线 > 故障）
- **AND** 超出布局容量的视频流应在列表中等待

#### Scenario: 视频流拖拽排序
- **WHEN** 用户拖拽某个视频窗口到另一个位置
- **THEN** 系统应交换两个窗口的视频流
- **AND** 保持两个视频流的播放状态
- **AND** 记录新的排序配置

### Requirement: OpenClaw AI智能助手集成
系统应集成OpenClaw AI助手，提供智能监控能力。

#### Scenario: AI对话分析
- **WHEN** 用户点击AI助手按钮或使用快捷键（Ctrl+K）
- **THEN** 系统应打开AI对话面板
- **AND** 用户可以询问"显示所有离线摄像头"、"分析最近1小时的告警趋势"等问题
- **AND** AI应调用相应工具返回结果，并在大屏上高亮显示相关视频流

#### Scenario: 智能告警分析
- **WHEN** 某路视频流触发YOLO检测告警（火灾、入侵等）
- **THEN** 系统应自动调用OpenClaw分析告警严重程度
- **AND** 根据分析结果自动调整该视频流的显示优先级
- **AND** 在AI面板中生成告警分析报告

#### Scenario: 语音控制
- **WHEN** 用户说出"切换到4画面布局"或"放大第3路视频"
- **THEN** 系统应识别语音指令并执行对应操作
- **AND** 在AI面板中显示执行结果

### Requirement: 视频流状态监控
系统应实时监控所有视频流的状态，并在界面上清晰展示。

#### Scenario: 状态指示器
- **WHEN** 视频流正常播放
- **THEN** 窗口边框显示绿色指示灯
- **AND** 显示实时帧率、码率、延迟等信息

#### Scenario: 异常状态提示
- **WHEN** 视频流出现异常（卡顿、延迟过高、信号弱）
- **THEN** 窗口边框显示黄色或红色指示灯
- **AND** 在窗口右上角显示异常类型图标
- **AND** 鼠标悬停显示详细信息

### Requirement: 性能优化
系统应优化多路视频流的性能，确保流畅播放。

#### Scenario: 自适应码率
- **WHEN** 同时播放的视频流数量超过9路
- **THEN** 系统应自动降低非焦点窗口的视频码率
- **AND** 焦点窗口保持高清画质
- **AND** 用户点击某个窗口时，该窗口自动成为焦点

#### Scenario: 虚拟滚动优化
- **WHEN** 用户滚动视频流列表
- **THEN** 系统应只渲染可见区域的视频窗口
- **AND** 离开视野的窗口应暂停播放以节省资源
- **AND** 重新进入视野时自动恢复播放

### Requirement: 快捷操作
系统应提供便捷的操作方式，提高监控效率。

#### Scenario: 双击全屏
- **WHEN** 用户双击某个视频窗口
- **THEN** 该窗口应全屏显示
- **AND** 其他窗口最小化到侧边栏
- **AND** 再次双击或按ESC退出全屏

#### Scenario: 快捷键操作
- **WHEN** 用户按下数字键1-5
- **THEN** 系统应切换到对应的布局模式（1=1x1, 2=2x2, ...）
- **WHEN** 用户按下F键
- **THEN** 当前焦点窗口全屏
- **WHEN** 用户按下Ctrl+K
- **THEN** 打开AI助手面板
