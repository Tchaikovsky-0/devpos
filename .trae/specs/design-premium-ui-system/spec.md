# 巡检宝高端UI设计系统规范

## Why

巡检宝作为企业级智能巡检监控平台，需要一套能够体现**专业性、科技感、高级感**的UI设计系统。当前UI虽然功能完善，但缺乏统一的高端视觉语言，无法充分传达产品的技术实力和品牌价值。

通过引入**Apple Pro + Linear + Vercel**的设计语言，打造一个让用户第一眼就感受到"这产品很专业"的视觉体验。

## What Changes

### 1. 视觉语言升级
- **深色主题优先**：采用深邃的深蓝灰色调，营造专业监控中心的氛围
- **玻璃拟态(Glassmorphism)**：半透明层叠效果，增加空间层次感
- **微光效果**： subtle glow 营造科技感和高级感
- **精致圆角**：统一的圆角系统，柔和但不失专业

### 2. 色彩系统重构
- 主色调：深邃科技蓝 `#16213e` → 强调蓝 `#4F7FD8`
- 背景层次：从深到浅的渐变层次系统
- 功能色：精心调校的成功、警告、错误色
- 渐变运用：微妙的渐变增加视觉深度

### 3. 排版系统优化
- 字体：Inter(英文) + Noto Sans SC(中文) 组合
- 字重：更细致的字重分级 (400/500/600)
- 行高：舒适的阅读体验 (1.5-1.6)
- 字号：更克制的字号层级

### 4. 动效系统
- **入场动画**：优雅的淡入上滑效果
- **交互动效**：微弹性反馈 (spring physics)
- **状态过渡**：流畅的状态切换
- **加载动画**：精致的骨架屏和加载指示器

### 5. 组件库升级
- **按钮**：多层阴影 + 悬浮提升效果
- **卡片**：微妙的边框 + 悬浮阴影
- **输入框**：聚焦发光效果
- **导航**：清晰的激活状态指示
- **数据展示**：专业的数据可视化组件

### 6. 布局系统
- **栅格系统**：12列响应式栅格
- **间距系统**：8px基准的和谐间距
- **容器**：最大宽度限制 + 自动居中
- **侧边栏**：可折叠的精致导航

## Impact

### 受影响模块
- 全局样式系统 (tailwind.config.js, design-system.css)
- 所有页面组件
- 侧边栏导航
- 数据大屏
- 表单组件
- 图表组件

### 用户体验提升
- 视觉层次更清晰
- 操作反馈更明确
- 专业感大幅提升
- 长时间使用不易疲劳

## ADDED Requirements

### Requirement: 深色主题设计系统

系统 SHALL 提供一套完整的深色主题设计系统：

#### Scenario: 色彩系统
- **GIVEN** 用户访问任意页面
- **WHEN** 页面渲染时
- **THEN** 使用统一的深色色彩系统：
  - 背景色：`#0d1117` (最深) → `#161b22` (默认) → `#21262d` (浅层)
  - 主色调：`#4F7FD8` (强调蓝)
  - 文字色：`#f0f6fc` (主文字) → `#8b949e` (次要) → `#6e7681` (辅助)
  - 边框色：`rgba(255,255,255,0.1)` → `rgba(255,255,255,0.2)` (悬浮)

#### Scenario: 玻璃拟态效果
- **GIVEN** 卡片或浮层组件
- **WHEN** 渲染时
- **THEN** 应用玻璃拟态效果：
  - 背景：`rgba(255,255,255,0.05)`
  - 模糊：`backdrop-blur-xl`
  - 边框：`rgba(255,255,255,0.1)`

### Requirement: 精致动效系统

系统 SHALL 提供流畅优雅的动效体验：

#### Scenario: 入场动画
- **GIVEN** 页面或组件首次加载
- **WHEN** 元素进入视口
- **THEN** 应用淡入上滑动画：
  - 初始状态：`opacity: 0, translateY: 20px`
  - 结束状态：`opacity: 1, translateY: 0`
  - 时长：`300ms`
  - 缓动：`cubic-bezier(0.4, 0, 0.2, 1)`

#### Scenario: 交互动效
- **GIVEN** 用户与可交互元素交互
- **WHEN** 悬浮或点击时
- **THEN** 提供微弹性反馈：
  - 悬浮：轻微上浮 + 阴影增强
  - 点击：轻微缩放 (0.98)
  - 过渡：200ms ease-out

### Requirement: 高级组件样式

系统 SHALL 提供高级感的组件样式：

#### Scenario: 按钮组件
- **GIVEN** 主要操作按钮
- **WHEN** 渲染时
- **THEN** 应用高级按钮样式：
  - 背景：主色调渐变
  - 阴影：`0 4px 14px rgba(79,127,216,0.3)`
  - 悬浮：上浮2px + 阴影增强
  - 圆角：`12px`

#### Scenario: 卡片组件
- **GIVEN** 内容卡片
- **WHEN** 渲染时
- **THEN** 应用高级卡片样式：
  - 背景：`#161b22`
  - 边框：`1px solid rgba(255,255,255,0.1)`
  - 悬浮：边框变亮 + 轻微阴影
  - 圆角：`16px`

### Requirement: 数据可视化风格

系统 SHALL 提供专业的数据展示风格：

#### Scenario: 监控数据展示
- **GIVEN** 数据大屏或仪表盘
- **WHEN** 展示监控数据时
- **THEN** 使用专业风格：
  - 数字：等宽字体 + 大字号
  - 状态指示：彩色圆点 + 文字
  - 图表：深色主题配色
  - 实时数据：脉冲动画指示

## 设计原则

### 1. 简约而不简单
- 去除多余装饰，保留必要元素
- 每个像素都有存在的意义
- 留白是设计的一部分

### 2. 层次清晰
- 通过阴影、透明度、色彩建立层次
- 重要信息突出，次要信息弱化
- 视觉引导用户注意力

### 3. 一致性
- 全平台统一的视觉语言
- 可预测的用户体验
- 减少认知负担

### 4. 响应式
- 适配各种屏幕尺寸
- 触控友好的交互区域
- 流畅的过渡动画
