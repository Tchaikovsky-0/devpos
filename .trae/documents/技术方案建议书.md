## 一、存储规划方案

### 1.1 存储容量估算

**场景假设**：20路 2K视频流

```yaml
单路视频数据量计算：
- 分辨率：2K (2560×1440)
- 帧率：25fps
- 码率：4Mbps (H.265压缩)
- 每小时：4Mbps × 3600s ÷ 8 = 1.8 GB/小时
- 每天：1.8 GB × 24 = 43.2 GB/天
- 每月：43.2 GB × 30 = 1.3 TB/月

20路视频总数据量：
- 每天：43.2 GB × 20 = 864 GB ≈ 0.84 TB
- 每月：1.3 TB × 20 = 26 TB
- 每年：26 TB × 12 = 312 TB
```

### 1.2 推荐存储方案

#### 分层存储架构

```
┌─────────────────────────────────────────┐
│           存储分层架构                    │
├─────────────────────────────────────────┤
│                                         │
│  ┌─────────────┐  ┌─────────────┐     │
│  │   热数据    │  │   温数据    │     │
│  │  (SSD/高效) │  │  (HDD/普通) │     │
│  │             │  │             │     │
│  │  最近7天   │  │  8-30天    │     │
│  │  约6TB     │  │  约60TB    │     │
│  └─────────────┘  └─────────────┘     │
│                                         │
│  ┌─────────────┐  ┌─────────────┐     │
│  │   冷数据    │  │   归档存储   │     │
│  │  (HDD/慢)  │  │  (低成本)   │     │
│  │             │  │             │     │
│  │  31-90天   │  │  90天+     │     │
│  │  约180TB   │  │  可选保留   │     │
│  └─────────────┘  └─────────────┘     │
│                                         │
└─────────────────────────────────────────┘
```

#### 推荐配置

```yaml
存储方案选项：

方案A：本地存储（推荐中小企业）
├── 热存储：2TB NVMe SSD (约1500元)
├── 温存储：8TB HDD × 4 = 32TB (约4000元)
├── 冷存储：8TB HDD × 24 = 192TB (约24000元)
├── 总成本：约29500元（一次性投入）
└── 月成本：电费约200元

方案B：云存储（推荐分布式部署）
├── 腾讯云COS标准存储：0.118元/GB/月
├── 每天864GB = 约102元/天 = 3060元/月
├── 优点：弹性扩展，无需维护
└── 缺点：长期成本较高

方案C：混合存储（推荐大型企业）
├── 本地热数据：2TB SSD
├── 本地温数据：32TB HDD
├── 云端冷存储：按需使用
└── 优点：平衡成本与性能
```

### 1.3 存储优化策略

```yaml
视频压缩策略：
├── 实时流：H.265编码，4Mbps码率
├── 存储流：H.265编码，2Mbps码率（降低码率存储）
├── 抽帧策略：重要时刻全帧率，日常抽帧
└── AI分析：使用低分辨率流进行检测

元数据分离：
├── 视频文件：对象存储（MinIO/COS）
├── 元数据库：PostgreSQL（录像索引、时间戳）
├── 分析结果：时序数据库（TimescaleDB）
└── 用户数据：PostgreSQL
```

**建议决策**：
- ✅ 推荐 **方案C：混合存储**，平衡成本与性能
- ✅ 视频存储使用 **H.265压缩**，降低50%存储空间
- ✅ 实施 **智能抽帧**，进一步节省空间
