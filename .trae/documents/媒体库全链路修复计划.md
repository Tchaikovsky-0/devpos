## 二、修复清单

### P0 - 严重问题（5 项）

#### C1: 上传闭包捕获过期 currentFolderId
- **文件**：`frontend/src/pages/MediaLibrary.tsx`
- **问题**：`processUpload` 的 `useCallback` 依赖 `currentFolderId`，用户切换文件夹后闭包中的值过期
- **修复**：创建上传任务时快照 `folderId` 存入 `UploadTask`，`processUpload` 从 task 读取而非闭包

#### C2: 上传无并发控制
- **文件**：`frontend/src/pages/MediaLibrary.tsx`
- **问题**：100 个文件同时发起请求
- **修复**：实现并发池，限制同时上传 3 个文件

#### C3: FolderPermission 缺少 tenant_id
- **文件**：`backend/internal/model/media.go`
- **问题**：权限表无 `tenant_id`，违反多租户隔离
- **修复**：
  1. 模型添加 `TenantID string` 字段 + 索引
  2. `CreateFolder` 中创建权限时传入 `tenant_id`
  3. 所有权限查询添加 `tenant_id` 过滤
  4. 数据库迁移：`ALTER TABLE folder_permissions ADD COLUMN tenant_id VARCHAR(64)`

#### C4: 权限操作无授权验证
- **文件**：`backend/internal/service/media.go`
- **问题**：任何用户可修改任何文件夹权限
- **修复**：`GrantFolderPermission`/`RevokeFolderPermission`/`UpdateFolderPermission` 中检查调用者是否为文件夹 admin

#### C5: 已删除/回收站文件仍可下载
- **文件**：`backend/internal/service/media.go` - `GetFilePath`
- **问题**：未检查 `trashed_at` 和 `deleted_at`
- **修复**：查询条件添加 `AND trashed_at IS NULL AND deleted_at IS NULL`

### 新功能 - 语义去重

#### 架构设计

```
用户上传/手动触发
    ↓
pHash 快速初筛（Python AI 新增接口）
    ↓ 相似度 > 阈值
OpenClaw 多模态精细判断
    ↓ 确认为同一缺陷
移入回收站（保留最佳质量的一张）
```

#### 后端实现

**1. Python AI 新增 pHash 接口**
- **文件**：`ai-service/app/main.py`
- **新增端点**：`POST /api/v1/image/phash`
- **输入**：图片文件
- **输出**：`{ phash: "hex_string", width: int, height: int, file_size: int }`
- **依赖**：`imagehash` + `Pillow`

**2. Go 新增语义去重服务**
- **文件**：`backend/internal/service/media.go`
- **新增函数**：`SemanticDedupe(tenantID string, ids []uint) (*SemanticDedupeResult, error)`
- **流程**：
  1. 查询指定文件，过滤非图片
  2. 逐个调用 Python AI 获取 pHash
  3. 计算汉明距离，相似度 > 0.85 的分组
  4. 每组调用 OpenClaw 多模态判断是否为同一缺陷
  5. 保留最佳质量（分辨率最高/文件最大）的一张
  6. 其余移入回收站

**3. Handler + Router**
- **文件**：`backend/internal/handler/media.go`
- **新增端点**：`POST /media/batch/semantic-dedupe`
- **请求体**：`{ "ids": [1,2,3], "auto_mode": false }`
- `auto_mode=true` 时上传后自动触发

**4. 上传后自动触发**
- **文件**：`backend/internal/service/media.go` - `UploadFile`
- 上传成功后，异步 goroutine 检查同文件夹内是否有相似图片
- 使用 goroutine + channel 避免阻塞上传响应

#### 前端实现

**1. 智能去重按钮**
- **文件**：`frontend/src/pages/MediaLibrary.tsx`
- 工具栏添加「智能去重」按钮
- 选中文件时去重选中文件，未选中时去重当前文件夹

**2. 去重结果展示**
- **文件**：`frontend/src/components/media/SemanticDedupeDialog.tsx`（新建）
- 展示 AI 识别的相似图片组
- 每组显示缩略图对比，标记推荐保留的文件
- 用户可调整保留/删除选择后确认执行

**3. API 层**
- **文件**：`frontend/src/store/api/mediaApi.ts`
- 新增 `semanticDedupe` mutation
