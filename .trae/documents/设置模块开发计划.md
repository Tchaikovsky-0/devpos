## 📋 一、项目背景与目标

### 1.1 背景
巡检宝作为面向重工业企业的智能监控平台，需要一个功能完善、操作简洁的设置模块，用于管理系统配置、用户偏好、告警规则、检测参数等。

### 1.2 设计理念
参考行业优秀产品（如 Grafana、Prometheus、Zabbix）的设计理念：
- **简洁直观**: 隐藏复杂配置，提供默认值
- **分层组织**: 按功能分类，逐级深入
- **实时反馈**: 修改即时生效，提供预览
- **安全防护**: 敏感操作二次确认

### 1.3 核心目标
1. ✅ 提供完整的系统设置功能
2. ✅ 界面简洁，易于操作
3. ✅ 隐藏用户不关心的配置项
4. ✅ 符合巡检宝设计系统规范

## 🏗️ 三、功能架构设计

### 3.1 设置模块功能树
```
系统管理 (/system)
├── 个人设置
│   ├── 主题切换 (深境/均衡/清境)
│   ├── 语言偏好
│   ├── 通知订阅
│   └── 快捷键设置
├── 告警配置
│   ├── 告警规则列表
│   ├── 规则创建/编辑
│   ├── 告警分级
│   └── 静默管理
├── 检测设置
│   ├── YOLO模型配置
│   ├── 置信度阈值
│   ├── 检测区域(ROI)
│   └── 防抖参数
├── 通知管理
│   ├── 渠道配置 (钉钉/企业微信/邮件)
│   ├── 通知模板
│   └── 发送频率
└── 系统信息
    ├── 版本信息
    ├── 资源监控
    ├── 日志查看
    └── 数据备份
```

### 3.2 隐藏的配置项
根据"隐藏用户不关心的配置"原则，以下配置默认隐藏：
- ❌ 技术底层参数（如数据库连接池大小）
- ❌ 调试选项（如日志级别 DEBUG）
- ❌ 性能调优参数（如缓存大小）
- ✅ 仅在"高级设置"中选择性暴露

## 🎨 六、UI/UX 设计规范

### 6.1 布局结构
```
┌─────────────────────────────────────────────────────────┐
│  系统管理                                             │
├──────────┬──────────────────────────────────────────┤
│          │  [设置项1]                              │
│  个人设置 │  ┌──────────────────────────────────┐   │
│  告警配置 │  │ 配置表单                        │   │
│  检测设置 │  │                                │   │
│  通知管理 │  └──────────────────────────────────┘   │
│  系统信息 │                                          │
│          │  [保存] [重置]                          │
└──────────┴──────────────────────────────────────────┘
```

### 6.2 设计原则
1. **分组导航**: 左侧导航按功能分组
2. **卡片布局**: 每个设置项使用独立卡片
3. **即时保存**: 修改自动保存，提供撤销
4. **视觉反馈**: 操作成功/失败明确提示
5. **渐进式披露**: 高级设置默认隐藏

## 🔧 七、技术实现要点

### 7.1 前端技术栈
- React 18 + TypeScript
- Tailwind CSS (遵循设计系统)
- React Router (路由管理)
- Zustand (状态管理)
- React Hook Form (表单处理)

### 7.2 后端技术栈
- Go 1.21+ + Gin
- PostgreSQL (配置存储)
- Redis (缓存)
- RESTful API 设计
