# 媒体库人工标注与系统优化计划

## 需求理解

### 当前问题
1. **标注方式单一**：目前只有 OpenClaw AI 分析和 YOLO 自动检测两种标注方式，缺少人工手动标注功能
2. **OpenClaw 掉线**：OpenClaw 服务不稳定，经常断开连接
3. **文件夹布局问题**：文件夹树/详情面板的 UI 布局存在显示问题

### 目标
1. 在媒体库中实现人工标注功能，支持用户手动添加标注信息
2. 提高 OpenClaw 服务的稳定性
3. 修复文件夹布局问题

## 实施计划

### 第一阶段：人工标注功能

#### 1.1 数据库设计
- 创建 `annotations` 表存储人工标注数据
- 字段设计：
  - `id` (主键)
  - `media_id` (关联媒体文件)
  - `user_id` (标注用户)
  - `tenant_id` (租户隔离)
  - `annotation_type` (标注类型：tag/region/note)
  - `content` (标注内容 JSON)
  - `bbox` (区域标注的边界框，可选)
  - `tags` (标签数组)
  - `notes` (文字备注)
  - `created_at`, `updated_at`, `deleted_at`

#### 1.2 后端实现
- 创建 `Annotation` 模型 (`backend/internal/model/annotation.go`)
- 创建 `AnnotationService` (`backend/internal/service/annotation.go`)
  - `CreateAnnotation` - 创建标注
  - `ListAnnotations` - 获取媒体标注列表
  - `UpdateAnnotation` - 更新标注
  - `DeleteAnnotation` - 删除标注
  - `GetAnnotationsByMedia` - 按媒体获取标注
- 创建 `AnnotationHandler` (`backend/internal/handler/annotation.go`)
  - `POST /annotations` - 创建标注
  - `GET /annotations?media_id=xxx` - 获取标注列表
  - `PUT /annotations/:id` - 更新标注
  - `DELETE /annotations/:id` - 删除标注
- 在 router.go 中注册路由

#### 1.3 前端实现
- 创建 `AnnotationPanel` 组件 (`frontend/src/components/media/AnnotationPanel.tsx`)
  - 显示当前媒体的所有标注
  - 支持添加新标注（标签、区域、备注）
  - 支持编辑/删除标注
  - 标注列表按时间排序
- 创建 `AnnotationEditor` 组件 (`frontend/src/components/media/AnnotationEditor.tsx`)
  - 标注类型选择（标签/区域/备注）
  - 标签输入（支持多选）
  - 区域标注（在图片上绘制矩形框）
  - 备注文本输入
- 创建 `AnnotationOverlay` 组件 (`frontend/src/components/media/AnnotationOverlay.tsx`)
  - 在图片/视频上叠加显示区域标注
  - 支持点击标注查看详情
- 在 `MediaSideDetail` 中集成 `AnnotationPanel`
- 更新 `mediaApi.ts` 添加标注相关 API 调用

### 第二阶段：OpenClaw 稳定性优化

#### 2.1 问题诊断

- 添加降级策略

#### 2.3 后端实现
- 在 `OpenClawService` 中添加健康检查方法
- 实现连接池和自动重连

#### 2.4 前端实现
- 在 `OpenClawCopilot` 组件中显示连接状态
- 添加重连按钮
- 显示服务不可用时的友好提示

## 技术细节

### 数据库迁移
```sql
CREATE TABLE annotations (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    media_id BIGINT NOT NULL,
    user_id BIGINT NOT NULL,
    tenant_id VARCHAR(64) NOT NULL,
    annotation_type VARCHAR(32) NOT NULL,
    content JSON,
    bbox JSON,
    tags JSON,
    notes TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at TIMESTAMP NULL,
    INDEX idx_media_id (media_id),
    INDEX idx_tenant_id (tenant_id),
    INDEX idx_user_id (user_id)
);
```

## 实施顺序

1. **数据库迁移** - 创建 annotations 表
2. **后端模型和服务** - Annotation 模型、Service、Handler
3. **后端路由** - 注册标注相关 API
4. **前端 API** - 更新 mediaApi.ts 添加标注 API
5. **前端组件** - AnnotationPanel、AnnotationEditor、AnnotationOverlay
6. **集成测试** - 测试标注功能的完整流程
7. **OpenClaw 优化** - 健康检查、重连机制、降级策略
8. **布局修复** - 修复文件夹树和详情面板的布局问题
9. **全面测试** - 端到端测试所有功能

## 风险评估

- **数据库变更**：需要执行迁移脚本，注意备份
- **前端兼容性**：新组件需要与现有组件兼容
- **性能影响**：标注数据可能增加查询复杂度，需要添加索引
- **用户体验**：需要确保标注功能易于使用，不影响现有工作流
