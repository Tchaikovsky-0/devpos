## 发现的小问题（非阻塞）

### 问题 1：vite.config.ts 中 `vendor-redux` 键重复 🔴 值得一修

**位置**：`frontend/vite.config.ts` 第 34 行和第 50 行

```typescript
manualChunks: {
  'vendor-react': ['react', 'react-dom', 'react-router-dom'],
  'vendor-redux': ['@reduxjs/toolkit', 'react-redux'],  // ← 第 34 行
  'vendor-ui': [ ... ],
  'vendor-motion': ['framer-motion', 'lucide-react'],
  'vendor-query': ['@tanstack/react-query'],
  'vendor-redux': ['@reduxjs/toolkit', 'react-redux'],  // ← 第 50 行，重复！
}
```

**影响**：
- 构建时会产生一个 WARNING（`Duplicate key "vendor-redux" in object literal`）
- JavaScript 对象字面量中后一个会覆盖前一个，但因为内容完全相同，实际无影响
- 不影响功能，但不够整洁

**修复方案**：删除第 50 行的重复 `vendor-redux` 条目

### 问题 2：默认路由可能不是 `/center`（观察到的轻微异常）

**现象**：浏览器首次打开 `http://localhost:5177/` 时，页面显示的是「航拍图库」而不是「监控中枢」

**原因分析**：
- 路由配置中 `path: ''` 对应 Center 组件，应该会匹配 `/`
- 但 `findModuleForPath('/')` 的 `normalizePath` 函数对 `/` 有特殊处理（返回 `/`），而 navigation.ts 中 center 模块的 `aliases` 包含 `/`
- 可能是浏览器缓存或 React Router 初始状态的问题

**实际影响**：极小，刷新或点击导航后即正常

## 修复步骤

### 步骤 1：修复 vite.config.ts 重复键
- 删除第 49-50 行的重复 `vendor-redux` 条目
- 保留第 33-34 行的原始定义即可
