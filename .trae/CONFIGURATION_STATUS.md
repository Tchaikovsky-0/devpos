## 🚀 使用指南

### 快速开始

1. **使用技能调度器** (推荐入口)
   ```
   /dispatch [任务描述]
   或
   @skill-dispatcher [任务描述]
   ```

2. **直接调用 Agent**
   ```
   @project-lead [决策请求]
   @frontend-dev [前端开发任务]
   @backend-dev [后端开发任务]
   @openclaw-eng [AI/Agent任务]
   ```

3. **使用快捷命令**
   ```
   /lead      - 项目总负责人
   /fe-lead   - 前端架构师
   /be-lead   - 后端架构师
   /ai-lead   - AI 技术负责人
   /devops    - DevOps 工程师
   /qa        - 测试负责人
   ```

### 最佳实践

1. **所有开发任务** 先通过 `skill-dispatcher` 分析任务类型
2. **复杂功能** 使用 Plan 模式进行规划
3. **新功能开发** 遵循 TDD 测试驱动开发
4. **代码修改** 必须通过 Code Review
5. **跨 Agent 协作** 遵循标准通信协议

## ⚠️ 重要提醒

### 禁止事项

```yaml
❌ 架构禁止:
  - Go 中直接调用 YOLO（必须通过 Python 服务）
  - Python 中处理 WebSocket 连接
  - 跨服务直接访问数据库
  - 绕过 API Gateway 直接访问服务

❌ 代码禁止:
  - 使用 any 类型（TypeScript）
  - 循环内数据库查询（N+1 问题）
  - 忽略 error 返回值
  - 硬编码敏感信息
  - SELECT * 查询
```

### 质量要求

```yaml
测试覆盖率：> 70%
API 响应 P95: < 200ms
前端首屏：< 3 秒
Lighthouse: > 90
```
