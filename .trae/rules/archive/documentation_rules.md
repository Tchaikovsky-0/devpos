## 📌 规范概述

本规范旨在为巡检宝项目的文档编写提供**统一、规范的方法**。核心目标是让知识不再流失，让文档成为资产。

---

## 🎯 核心原则

### 1. 文档优先级

```yaml
优先级排序:
  1️⃣ API文档 (如何使用)
  2️⃣ 架构文档 (如何设计)
  3️⃣ 开发指南 (如何开发)
  4️⃣ 运维文档 (如何运维)

核心原则:
  ✅ 代码即文档
  ✅ 文档同步
  ✅ 简洁明了
  ✅ 实用为主
```

---

## 📋 文档规则

### R1: 文档类型规则

**规则描述**：不同类型的文档有不同要求。

**文档类型**：

```yaml
API文档:
  ✅ 端点描述
  ✅ 请求参数
  ✅ 响应格式
  ✅ 使用示例

架构文档:
  ✅ 系统概览
  ✅ 架构设计
  ✅ 数据模型
  ✅ 调用流程

开发指南:
  ✅ 快速开始
  ✅ 项目结构
  ✅ 开发流程
  ✅ 测试指南
```

---

### R2: 文档质量规则

**规则描述**：文档必须高质量。

**质量标准**：

```yaml
完整性:
  ✅ 覆盖所有功能
  ✅ 包含示例
  ✅ 解答常见问题

准确性:
  ✅ 内容正确
  ✅ 示例可运行
  ✅ 与代码一致

可读性:
  ✅ 结构清晰
  ✅ 语言简洁
  ✅ 格式规范
```

---

### R3: 文档维护规则

**规则描述**：文档必须与代码同步更新。

**维护机制**：

```yaml
同步要求:
  ✅ 代码变，文档必须变
  ✅ PR包含文档更新
  ✅ 文档审查

定期检查:
  ✅ 季度文档审计
  ✅ 删除过时文档
  ✅ 更新过期内容
```

---

## 🚫 禁止规则

### R4: 文档禁止规则

```yaml
🚫 绝对禁止:
  - 文档与代码不一致
  - 过时的文档不更新
  - 代码没有注释
  - API没有文档

⚠️ 强烈不推荐:
  - 长段落不分段
  - 没有示例
  - 明显的代码不写注释
  - 过期的注释
```
