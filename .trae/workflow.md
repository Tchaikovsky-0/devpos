# 巡检宝 - 多Agent协作工作流

> 本文档定义了多Agent之间的协作流程、任务分发规则和沟通协议。

---

## 一、协作架构

### 1.1 团队层级

```
                    ┌─────────────────┐
                    │  Project Lead   │  ← 决策层
                    │   (决策/协调)    │
                    └────────┬────────┘
                             │
        ┌────────────────────┼────────────────────┐
        │                    │                    │
        ▼                    ▼                    ▼
┌───────────────┐  ┌───────────────┐  ┌───────────────┐
│ Frontend Lead │  │ Backend Lead │  │   AI Lead    │  ← Lead层
│  (前端架构)    │  │  (后端架构)   │  │  (AI架构)    │
└───────┬───────┘  └───────┬───────┘  └───────┬───────┘
        │                    │                    │
        ▼                    ▼                    ▼
┌───────────────┐  ┌───────────────┐  ┌───────────────┐
│ Frontend Dev  │  │ Backend Dev  │  │ OpenClaw Eng │  ← Dev层
│  (前端开发)    │  │  (后端开发)   │  │ (Agent开发)  │
└───────────────┘  └───────────────┘  └───────────────┘

        ┌────────────────────┴────────────────────┐
        │                                         │
        ▼                                         ▼
┌───────────────┐                       ┌───────────────┐
│  DevOps Eng   │                       │   QA Lead    │  ← 支撑层
│  (运维部署)    │                       │  (测试质量)  │
└───────────────┘                       └───────────────┘
```

### 1.2 职责边界

| 层级 | 职责 | 决策权 |
|------|------|--------|
| 决策层 | 战略规划、架构审批、资源分配 | ✅ 最终决策 |
| Lead层 | 技术方案、任务分配、代码审查 | ✅ 领域决策 |
| Dev层 | 功能实现、Bug修复、单元测试 | ❌ 执行 |
| 支撑层 | 环境管理、质量保障、CI/CD | ✅ 专业决策 |

## 二、任务分发流程

### 2.1 新功能开发流程

```
┌─────────────────────────────────────────────────────────────┐
│                    新功能开发工作流                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 需求分析 (Project Lead)                                 │
│     └── 理解需求、评估复杂度、确定优先级                      │
│                                                             │
│  2. 架构设计 (相关 Lead)                                     │
│     ├── 前端架构 → Frontend Lead                            │
│     ├── 后端架构 → Backend Lead                             │
│     └── AI架构 → AI Lead                                    │
│                                                             │
│  3. 任务拆分 (Project Lead)                                  │
│     └── 生成任务列表、分配责任人、设置依赖关系                 │
│                                                             │
│  4. 并行开发                                                 │
│     ├── 后端开发 → Backend Dev (Backend Lead 指导)          │
│     ├── 前端开发 → Frontend Dev (Frontend Lead 指导)        │
│     └── AI开发 → OpenClaw Eng (AI Lead 指导)                │
│                                                             │
│  5. 集成测试 (QA Lead)                                       │
│     └── 功能测试、集成测试、性能测试                          │
│                                                             │
│  6. 部署上线 (DevOps Eng)                                    │
│     └── 环境准备、部署配置、监控告警                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 任务分发规则

```yaml
任务类型分发规则:

  前端任务:
    简单组件: Frontend Dev (Frontend Lead 审查)
    复杂组件: Frontend Lead 开发 + 审查
    架构变更: Frontend Lead 设计 → Project Lead 审批

  后端任务:
    简单API: Backend Dev (Backend Lead 审查)
    复杂业务: Backend Lead 开发 + 审查
    架构变更: Backend Lead 设计 → Project Lead 审批

  AI任务:
    模型集成: AI Lead 负责
    Agent开发: OpenClaw Eng (AI Lead 指导)
    工具开发: OpenClaw Eng

  运维任务:
    环境配置: DevOps Eng
    CI/CD: DevOps Eng
    监控告警: DevOps Eng

  测试任务:
    测试计划: QA Lead
    测试执行: QA Lead
    Bug管理: QA Lead
```

## 四、沟通协议

### 4.1 状态报告格式

```markdown
## [Agent名称] 工作状态 - YYYY-MM-DD

### 已完成 ✅
- [x] 任务1 - 简要说明
- [x] 任务2 - 简要说明

### 进行中 🔄
- [ ] 任务3 - 进度: 80%
- [ ] 任务4 - 进度: 50%

### 阻塞 ⚠️
- 问题1 - 需要xxx配合
- 问题2 - 等待xxx决策

### 下一步 📋
- [ ] 任务5
- [ ] 任务6
```

### 4.2 问题升级路径

```
┌─────────────────────────────────────────────────────────────┐
│                      问题升级流程                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  P3 - 轻微问题                                               │
│  └── Dev Agent 自行处理                                      │
│      └── 记录到日志                                          │
│                                                             │
│  P2 - 一般问题                                               │
│  └── Dev Agent → Lead Agent                                 │
│      └── Lead 协调解决                                       │
│                                                             │
│  P1 - 严重问题                                               │
│  └── Lead Agent → Project Lead                              │
│      └── 24小时内解决                                        │
│                                                             │
│  P0 - 阻断问题                                               │
│  └── 立即升级 → Project Lead                                 │
│      └── 立即处理，暂停其他任务                               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 五、代码审查流程

### 5.1 审查权限

| 代码类型 | 一级审查 | 二级审查 |
|----------|----------|----------|
| 前端代码 | Frontend Lead | Project Lead (重大变更) |
| 后端代码 | Backend Lead | Project Lead (重大变更) |
| AI代码 | AI Lead | Project Lead (重大变更) |
| 配置文件 | DevOps Eng | Project Lead |
| 测试代码 | QA Lead | - |

### 5.2 审查清单

```markdown
## Code Review 清单

### 功能正确性
- [ ] 代码逻辑正确
- [ ] 边界条件处理
- [ ] 错误处理完善

### 代码质量
- [ ] 命名规范
- [ ] 代码可读性
- [ ] 注释完整

### 性能
- [ ] 无性能问题
- [ ] 资源使用合理

### 安全
- [ ] 无安全漏洞
- [ ] 敏感信息处理

### 测试
- [ ] 单元测试覆盖
- [ ] 测试用例合理

### 规范
- [ ] 遵循项目规范
- [ ] 无禁止代码
```

## 七、应急响应

### 7.1 故障等级定义

| 等级 | 定义 | 响应时间 | 处理时限 |
|------|------|----------|----------|
| P0 | 核心功能不可用 | 5分钟 | 1小时 |
| P1 | 重要功能受损 | 15分钟 | 4小时 |
| P2 | 一般功能问题 | 1小时 | 24小时 |
| P3 | 轻微问题 | 4小时 | 72小时 |

### 7.2 应急响应流程

```
故障发现
    │
    ▼
DevOps Eng 初步判断
    │
    ├── 前端问题 → Frontend Lead
    ├── 后端问题 → Backend Lead
    ├── AI问题 → AI Lead
    └── 基础设施 → DevOps Eng
    │
    ▼
问题定位 & 修复
    │
    ▼
QA Lead 验证
    │
    ▼
Project Lead 确认恢复
    │
    ▼
故障复盘
```
