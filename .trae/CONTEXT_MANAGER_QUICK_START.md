## 🚀 快速使用

### 自动触发（无需手动调用）

Context Manager 会在以下情况**自动触发**：

1. **上下文达到 L2 警戒级别** (50-100条消息)
   - 我会自动提醒你上下文开始变长
   - 准备执行摘要操作

2. **上下文达到 L3 警告级别** (100-200条消息)
   - 我会强制要求生成摘要
   - 必须清理后才能继续

3. **每执行 5 个操作后**
   - 我会执行规则守护程序检查
   - 确保没有规则被忽略

4. **任务完成或进入新阶段时**
   - 我会生成任务摘要
   - 清理冗余上下文

### 手动触发

你可以随时手动调用：

```bash
# 检查上下文健康状态
/context

# 生成对话摘要
@context-manager 生成摘要

# 清理上下文
/ctx 清理冗余
```

## 📊 上下文健康级别

| 级别 | 消息数 | 状态 | 建议操作 |
|------|--------|------|---------|
| **L1** | 0-50 | ✅ 完美 | 畅快执行，无需干预 |
| **L2** | 50-100 | ⚠️ 警戒 | 开始摘要，准备清理 |
| **L3** | 100-200 | 🔴 警告 | 必须摘要后才能继续 |
| **L4** | >200 | 🚨 危险 | 暂停任务，立即摘要 |
| **L5** | >500 | 💥 崩溃 | 立即重置 |

## 🎯 核心功能

### 1. 智能摘要生成

当需要摘要时，我会生成包含以下内容的结构化摘要：

```
# 📋 对话摘要 - [时间戳]

## 任务目标 🎯
[用户最初想要完成的目标]

## 已完成工作 ✅
1. ✅ [完成的任务1]
2. ✅ [完成的任务2]

## 当前状态 📍
- 正在执行：[当前任务]
- 进度：60% / Phase 2/3
- 性能状态：L2 警戒 ⚠️

## 待完成任务 📝
- [ ] 任务A - [预期动作]
- [ ] 任务B - [预期动作]

## 关键信息 🔑
- **涉及文件**：[主要修改的文件]
- **技术决策**：[做出的重要选择]
- **用户约定**：[对用户的承诺]

## 下一步行动 ▶️
1. [具体动作1]
2. [具体动作2]
```

### 2. 规则守护程序

每5个操作我会执行一次检查：

```
🛡️ 规则守护程序检查

## 1. 幻觉防范检查 🔍
- [✅] 最近提供的信息是否都有来源标注？
- [✅] 是否有未标注的推测内容？
- [✅] 是否有无法验证的断言？

## 2. 上下文长度检查 📏
- [✅] 当前上下文长度级别？ → L2 警戒
- [✅] 是否需要生成摘要？ → 是的
- [✅] 是否有信息丢失的风险？ → 低风险

## 3. 任务进度检查 📊
- [✅] 是否偏离原始目标？ → 没有
- [✅] 是否需要重新确认用户意图？ → 不需要

总体评估：✅ 可以继续，但建议生成摘要
```

### 3. 主动预警系统

**性能下降预警**：

```
⚠️ 性能预警

我注意到上下文正在变得冗长，这可能导致我的表现下降。

当前状态：
- 上下文长度：L2 警戒
- 性能指标：⚠️ 轻微下降

建议措施：
1. [ ] 生成对话摘要
2. [ ] 确认关键信息
3. [ ] 清理冗余内容

您希望我现在生成摘要吗？
```

## 💡 使用场景

### 场景 1: 长时间对话后

**问题**: 对话进行了很久，感觉变慢了

**解决**:
```bash
# 直接让我检查
/context

# 或者
@context-manager 检查上下文健康状态
```

我会：
1. 检查当前上下文长度
2. 生成摘要
3. 清理冗余
4. 确认关键信息

### 场景 2: 任务切换

**问题**: 需要从任务A切换到任务B

**解决**:
```bash
# 让我先总结任务A
@context-manager 生成任务A摘要

# 然后清理上下文，开始任务B
/ctx 清理上下文
```

### 场景 3: 感觉回复质量下降

**问题**: 感觉我开始重复问题或遗漏信息

**解决**:
```bash
# 让我执行规则守护程序
@context-manager 运行规则守护程序

# 生成摘要，刷新上下文
@context-manager 生成完整摘要
```

## 🎓 最佳实践

### ✅ 应该做的

1. **定期检查**
   - 长任务中间主动检查一次
   - 任务完成时生成摘要

2. **主动预警**
   - 我会主动提醒你上下文状态
   - 收到预警时及时处理

3. **任务分块**
   - 大任务拆成小任务
   - 每块任务后生成摘要

### ❌ 不应该做的

1. **不要等到崩溃**
   - L4/L5 级别很难恢复
   - L2/L3 时就要处理

2. **不要忽略预警**
   - 我的预警都是预防性的
   - 忽视会导致性能下降

3. **不要一次性太多任务**
   - 一次做太多会让上下文爆炸
   - 分批处理更高效


## 🎯 总结

**Context Manager 的核心职责**：

1. 🎯 **保持上下文健康** - 监控长度，定期清理
2. 🛡️ **守护规则执行** - 防止规则被淹没
3. ⚡ **保持高效执行** - 预防性能下降
4. ⚠️ **主动预警** - 不等问题发生
5. 🔄 **快速恢复** - 从问题中快速恢复

**记住**：
- 上下文就像手机电量，用一点少一点
- 摘要就像充电宝，让你的"脑子"重新满血
- 规则守护就像安全带，关键时刻保护你
