## 三、巡检宝项目部署方案

巡检宝采用 Docker Compose 容器化部署，共 7 个核心服务 + 4 个监控服务：

```
                     ┌──────────────┐
                     │   Nginx      │  (宿主机，端口 80/888)
                     │   反向代理    │
                     └──────┬───────┘
                            │
        ┌───────────────────┼───────────────────┐
        │                   │                   │
  ┌─────▼─────┐      ┌─────▼─────┐      ┌──────▼──────┐
  │ Frontend  │      │ Backend   │      │ OpenClaw    │
  │ React     │      │ Go API    │      │ AI Agent    │
  │ :3000→80  │      │ :8094     │      │ :8096       │
  └───────────┘      └─────┬─────┘      └─────────────┘
                          │
              ┌───────────┼───────────┐
              │           │           │
        ┌─────▼─────┐ ┌──▼──┐  ┌────▼─────┐
        │ AI Service│ │Redis│  │   YOLO   │
        │ Python    │ │:6379│  │ Service  │
        │ :8095     │ └─────┘  │  :8097   │
        └───────────┘          └──────────┘
              │
        ┌─────▼─────┐
        │  MySQL    │
        │  :3306    │
        └───────────┘
```

### 3.2 资源分配方案（基于 15GB 内存）

由于实际内存为 15GB 而非原报告假设的 8GB，可以给各服务分配更多资源：

| 服务 | 内存限制 | CPU 限制 | 内存预留 | 说明 |
|------|---------|---------|---------|------|
| MySQL | 3G | 2 核 | 1.5G | 核心数据层，buffer_pool 优先 |
| Redis | 1G | 0.5 核 | 512M | 缓存 + 会话存储 |
| Backend (Go) | 1.5G | 1 核 | 512M | API 服务，Go 内存效率高 |
| AI Service (Python) | 2.5G | 1.5 核 | 1G | YOLO 推理 + AI 服务 |
| YOLO Service | 3G | 2 核 | 2G | 模型加载 + 推理（CPU 模式需更多内存） |
| OpenClaw | 768M | 0.5 核 | 384M | AI Agent 网关 |
| Frontend (Nginx) | 256M | 0.25 核 | 128M | 静态资源服务 |
| Prometheus | 768M | 0.5 核 | 384M | 监控采集 |
| Grafana | 384M | 0.25 核 | 192M | 监控面板 |
| Loki | 384M | 0.25 核 | 192M | 日志聚合 |
| Promtail | 192M | 0.15 核 | 96M | 日志采集 |
| **合计** | **~13.7G** | **~8.9 核** | **~6G** | |

> **注意**: CPU vCore 为超分配置，实际物理 4 核，Docker 会按权重调度。建议在生产环境中根据实际负载逐步调整。

### 3.3 推荐的 docker-compose 资源调整

基于 15GB 实际内存，建议对现有 `docker-compose.prod.yml` 做以下调整：

**MySQL**: `innodb_buffer_pool_size` 从 1G 提升至 2G  
**Redis**: `maxmemory` 从 512MB 提升至 1GB  
**AI Service**: 内存限制从 2G 提升至 2.5G  
**YOLO Service**: 内存限制从 4G 降至 3G（非 GPU 模式下无需 4G）  
**OpenClaw**: 内存限制从 512M 提升至 768M  

调整后总内存占用约 13.7G，预留约 1.3G 给操作系统内核和突发缓冲。

## 四、系统优化建议

### 4.1 内核参数调优

创建 `/etc/sysctl.d/99-xunjianbao.conf`：

```bash
# 网络连接参数
net.core.somaxconn = 65535
net.core.netdev_max_backlog = 65535
net.ipv4.tcp_max_syn_backlog = 65535

# TCP 连接优化
net.ipv4.tcp_fin_timeout = 30
net.ipv4.tcp_keepalive_time = 600
net.ipv4.tcp_keepalive_probes = 3
net.ipv4.tcp_keepalive_intvl = 10
net.ipv4.tcp_max_tw_buckets = 262144
net.ipv4.tcp_tw_reuse = 1

# TCP 内存缓冲区
net.core.rmem_default = 262144
net.core.rmem_max = 16777216
net.core.wmem_default = 262144
net.core.wmem_max = 16777216
net.ipv4.tcp_rmem = 4096 87380 16777216
net.ipv4.tcp_wmem = 4096 65536 16777216

# 文件描述符
fs.file-max = 2097152
fs.nr_open = 2097152

# 虚拟内存（15GB 内存适用）
vm.swappiness = 10
vm.dirty_ratio = 15
vm.dirty_background_ratio = 5

# TCP BBR 拥塞控制
net.core.default_qdisc = fq
net.ipv4.tcp_congestion_control = bbr
```

执行 `sysctl --system` 生效。

### 5.2 防火墙规则

建议使用 `ufw` 配置：

```bash
# 仅允许必要端口
ufw default deny incoming
ufw default allow outgoing
ufw allow 80/tcp          # HTTP
ufw allow 443/tcp         # HTTPS
ufw allow 888/tcp         # Nginx 备用端口
ufw allow 29632/tcp       # SSH
ufw allow 8094/tcp        # Backend API
ufw allow 8096/tcp        # OpenClaw
ufw allow from 127.0.0.1 to any port 3306  # MySQL 仅本地
ufw allow from 127.0.0.1 to any port 6379  # Redis 仅本地
ufw enable
```

### 5.3 安全基线检查

```bash
# 1. 检查 SSH 配置
grep -E "^PermitRootLogin|^PasswordAuthentication|^PubkeyAuthentication" /etc/ssh/sshd_config

# 2. 检查 Redis 密码
redis-cli -a <password> ping

# 3. 检查 MySQL 远程访问
mysql -u root -p -e "SELECT user, host FROM mysql.user WHERE user NOT IN ('mysql.sys','mysql.session','mysql.infoschema','debian-sys-maint');"

# 4. 检查 Docker 容器特权模式
docker inspect --format='{{.HostConfig.Privileged}}' $(docker ps -q)
```

```bash
# /opt/scripts/daily-maintenance.sh

# Docker 日志清理

# 日志轮转
find /var/log -name "*.log" -mtime +7 -delete

if [ "$MEM_USAGE" -gt 85 ]; then
    echo "[WARNING] 内存使用率: ${MEM_USAGE}%"
fi
```
